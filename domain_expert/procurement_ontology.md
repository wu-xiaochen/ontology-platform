# 采购供应链领域本体设计

## 概述

本文档定义采购供应链领域的核心本体模型，用于指导知识库构建和智能化应用。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 供应商 | Supplier | 提供物料或服务的企业主体 | 供应商编码、名称、等级、资质、联系方式 |
| 采购商 | Buyer | 采购商品或服务的企业主体 | 企业编码、名称、行业、规模 |
| 物料 | Material | 采购或供应的物品 | 物料编码、名称、规格、类别、单位 |
| 采购订单 | Purchase Order | 采购活动的正式书面确认 | 订单编号、日期、数量、单价、金额 |
| 合同 | Contract | 采购双方达成的法律协议 | 合同编号、类型、有效期、条款 |
| 仓库 | Warehouse | 存储物料的设施 | 仓库编码、位置、容量、类型 |
| 库存 | Inventory | 仓库中物料的实际数量 | 物料ID、仓库ID、数量、批次 |
| 物流 | Logistics | 物料从供应商到采购商的运输过程 | 物流单号、运输方式、状态、轨迹 |

### 1.2 关系定义

```
供应商 ⊚ 供应 [物料] (多对多)
供应商 ⊚ 签订 [合同] (一对多)
采购商 ⊚ 下达 [采购订单] (一对多)
采购订单 ⊚ 关联 [物料] (多对一)
采购订单 ⊚ 触发 [物流] (一对多)
仓库 ⊚ 存储 [库存] (一对多)
物料 ⊚ 属于 [物料类别] (多对一)
```

### 1.3 物料类别体系

```
物料
├── 原材料 (Raw Material)
│   ├── 金属材料
│   ├── 化工原料
│   └── 农产品
├── 半成品 (Semi-finished Product)
├── 成品 (Finished Product)
├── 辅料 (Auxiliary Material)
├── 设备 (Equipment)
│   ├── 生产设备
│   └── 办公设备
└── 服务 (Service)
    ├── 物流服务
    ├── 咨询服务
    └── 维修服务
```

## 二、规律 (Laws)

### 2.1 供应链牛鞭效应

**定义**: 需求信息在供应链各环节传递时，逐级放大导致上游波动远大于下游的现象。

**公式**:
```
波动放大系数 = 上游需求方差 / 下游需求方差
```

**应对原则**:
- 信息共享：打通供应链各环节数据
- 缩短lead time：缩短采购和交付周期
- 稳定需求：建立长期合作关系

### 2.2 库存周转规律

**定义**: 库存从入库到出库的循环效率。

**核心指标**:
```
库存周转率 = 期间出库成本 / 平均库存
周转天数 = 365 / 周转率
```

**行业基准**:
- 快消品: 8-12次/年
- 制造业: 4-6次/年
- 原材料: 2-4次/年

### 2.3 供应商准时交付率 (OTD)

**定义**: 供应商在承诺日期内完成交付的比例。

**计算公式**:
```
OTD = 准时交付订单数 / 总订单数 × 100%
```

**等级划分**:
- 优秀: OTD ≥ 98%
- 良好: 95% ≤ OTD < 98%
- 合格: 90% ≤ OTD < 95%
- 不合格: OTD < 90%

### 2.4 采购价格弹性

**定义**: 采购量变化对采购价格的影响程度。

**规律**:
- 采购量与单价呈负相关
- 长期合作价格低于短期采购
- 集中采购可获得更高折扣

## 三、业务规则 (Rules)

### 3.1 供应商准入规则

| 条件 | 要求 |
|------|------|
| 资质要求 | 具备营业执照、相关行业许可证 |
| 质量体系 | 通过ISO 9001或同等质量体系认证 |
| 财务健康 | 资产负债率 < 70%，近两年盈利 |
| 产能要求 | 可满足订单峰值150%的产能 |
| 环保合规 | 符合当地环保法规要求 |

### 3.2 采购审批权限

| 采购金额 | 审批人 |
|----------|--------|
| < ¥10,000 | 采购员 |
| ¥10,000 - ¥50,000 | 采购经理 |
| ¥50,000 - ¥200,000 | 采购总监 |
| ¥200,000 - ¥1,000,000 | 副总经理 |
| > ¥1,000,000 | 总经理/董事会 |

### 3.3 采购流程规则

```
需求提出 → 供应商选择 → 询价比价 → 合同签订 → 订单下达 → 跟踪交付 → 验收入库 → 发票结算
```

**关键节点**:
1. **询价比价**: 询价供应商数量 ≥ 3家
2. **合同签订**: 金额 > ¥50,000 必须签订正式合同
3. **验收入库**: 质量检验合格后方可入库
4. **发票结算**: 发票、合同、验收单三单匹配

### 3.4 供应商绩效考核规则

**考核维度及权重**:

| 维度 | 权重 | 指标 |
|------|------|------|
| 质量 | 30% | 来料不良率、返工率、客诉次数 |
| 交付 | 30% | OTD准时率、订单变更响应速度 |
| 价格 | 20% | 与市场价格对比、年降幅度 |
| 服务 | 10% | 响应速度、技术支持能力 |
| 环保 | 10% | 环保合规、可持续发展 |

**等级评定**:
- A级 (≥90分): 战略供应商，优先合作
- B级 (75-89分): 合格供应商，正常合作
- C级 (60-74分): 观察供应商，辅导改进
- D级 (<60分): 不合格供应商，淘汰或替换

### 3.5 安全库存规则

**计算公式**:
```
安全库存 = (最大日用量 × 最长采购周期) - (平均日用量 × 平均采购周期)
```

**最低库存警示**:
- 触发补货: 库存 ≤ 安全库存
- 紧急补货: 库存 ≤ 安全库存 × 50%
- 断货风险: 库存 = 0

### 3.6 合同履约规则

- 付款账期: 默认月结30天，可协商
- 违约责任: 延迟交付按合同金额1%/天收取违约金
- 价格保护: 合同期内市场降价需同步调整
- 续约提前: 合同到期前60天启动续约评估

## 六、核心业务场景 (MVP)

### 6.1 供应商风险管理

**场景**: 自动评估供应商风险等级，触发预警

**规则**:
```
IF 供应商绩效评分 < 60 THEN 标记为高风险
IF 供应商交付准时率 < 90% THEN 触发预警
IF 供应商财务异常 THEN 暂停合作并通知采购经理
```

### 6.2 智能询报价

**场景**: 自动向多个供应商发送询价请求

**规则**:
```
IF 采购金额 > 50000 THEN 询价供应商数量 >= 3
IF 物料类别 = 战略物资 THEN 只向A级供应商询价
```

### 6.3 库存优化建议

**场景**: 基于历史消耗数据给出补货建议

**规则**:
```
IF 当前库存 <= 安全库存 THEN 建议补货
IF 预测需求增长 > 20% THEN 建议提前备货
```

## 七、聚量采购（Volume Pooling）⭐ 新增 2026-03-19

### 7.1 概念定义

**聚量采购**是通过整合多个采购方的需求，形成更大采购量，以获得更优价格和交付条件的采购策略。

**三种聚量级别**：

| 级别 | 英文 | 定义 | 适用条件 |
|------|------|------|---------|
| 全国聚量 | National Volume Pooling | 跨区域大体量集中采购 | 高价值、标准化、需求广泛 |
| 区域聚量 | Regional Volume Pooling | 同区域内多采购方联合 | 区域内需求一致、规模效应 |
| 预测性聚量 | Predictive Volume Pooling | 快速增长需求提前聚量 | 需求增速 > 40%/月，有爆发潜力 |

---

### 7.2 核心指标

#### 物料离散率（Material Diversity Ratio）
```
标准化分 = 物料种类数 ÷ 交易总次数
```
**含义**：每笔交易平均使用的物料 SKU 数。越低表示采购越集中、越标准化。

| 分数区间 | 标准化等级 | 聚量适用性 |
|---------|----------|----------|
| < 0.1 | 极高标准化 | 极适合聚量，可用固定 SKU 谈价 |
| 0.1 ~ 0.2 | 高标准化 | 适合全国聚量 |
| 0.2 ~ 0.3 | 一般标准化 | 适合区域/预测聚量 |
| > 0.5 | 高度离散 | 不适合聚量，高度定制化 |

#### 聚量 SKU 上限
```
品类内物料种类 < 100 种
```
超过 100 种 SKU 的品类被视为"高度定制化"，不适合聚量谈判。

---

### 7.3 三级聚量规则

#### 全国聚量（National Volume Pooling）
| 指标 | 阈值 | 说明 |
|------|------|------|
| 总金额 | > 1,000,000 CNY | 单品类年采购金额 |
| 采购商 | > 3 跨区域实体 | 不同地区/公司 |
| 交易频次 | ≥ 20 次 | 数据量需有统计意义 |
| 物料 SKU | < 100 种 | 标准化要求 |
| 标准化分 | < 0.2 | 物料离散率阈值 |

#### 区域聚量（Regional Volume Pooling）
| 指标 | 阈值 | 说明 |
|------|------|------|
| 总金额 | > 500,000 CNY | 单品类年采购金额 |
| 采购商 | > 2 同区域实体 | 同一省份/地区 |
| 交易频次 | ≥ 5 次 | 数据量需有统计意义 |
| 物料 SKU | < 100 种 | 标准化要求 |
| 标准化分 | < 0.3 | 物料离散率阈值 |

#### 预测性聚量（Predictive Volume Pooling）
| 指标 | 阈值 | 说明 |
|------|------|------|
| 总金额 | > 100,000 CNY | 单品类年采购金额 |
| 采购商 | > 2 跨区域实体 | 不同地区/公司 |
| 交易频次 | ≥ 5 次 | 数据量需有统计意义 |
| 物料 SKU | < 100 种 | 标准化要求 |
| 标准化分 | < 0.3 | 物料离散率阈值 |
| **环比增长** | **> 40%/月** | 近2个月金额增速（核心差异） |

### 7.5 聚量采购推理规则

#### 规则1：全国聚量候选识别
```
IF 总金额 > 100万
AND 跨区域采购商数 > 3
AND 交易次数 >= 20
AND 物料种类 < 100
AND (物料种类/交易次数) < 0.2
THEN 标记为"全国聚量候选"
```

#### 规则2：预测性聚量识别
```
IF 近2个月环比增长率 > 40%
AND 总金额 > 10万
AND 跨区域采购商数 > 2
AND 交易次数 >= 5
AND 标准化分 < 0.3
THEN 标记为"预测性聚量候选"（增长强劲，跟踪是否升级为全国聚量）
```

#### 规则3：聚量节省估算
```
聚量节省估算 = 当前总金额 × 预估降价比例
参考降价比例：
  - 全国聚量：5-15%（大批量折扣）
  - 区域聚量：3-8%（区域协同折扣）
```

### 7.6 聚量采购决策流程

```
Step 1: 数据收集 → 历史采购记录（金额、频次、物料、采购商）
         │
         ▼
Step 2: 计算指标 → 标准化分、物料 SKU 数、月度金额趋势
         │
         ▼
Step 3: 三级筛选 → 依次判断：全国 → 区域 → 预测性
         │
         ▼
Step 4: 优先级排序 → 按 (金额 × 标准化分改善潜力) 排序
         │
         ▼
Step 5: 供应商尽调 → Top3 集中度分析、竞争格局评估
         │
         ▼
Step 6: 聚量谈判 → 确定 SKU 清单、采购量、目标价格
```
