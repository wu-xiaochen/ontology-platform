# BMAD Help Skill - Qoder 适配版

## 简介

这是 BMAD-METHOD 方法论的 Qoder 适配版本。BMAD (Build More Architect Dreams) 是一个 AI 驱动的敏捷开发框架，提供结构化工作流和专门代理。

## 使用方法

在对话中引用此技能获取 BMAD 方法论指导。

---

# BMAD Method - 核心工作流

## 阶段 1: 分析 (Analysis)

### 任务分解清单
- [ ] 识别问题域和范围
- [ ] 分析现有代码结构
- [ ] 确定技术栈和依赖
- [ ] 识别风险和约束
- [ ] 定义成功标准

### 输出物
- 问题分析文档
- 技术栈清单
- 风险评估

## 阶段 2: 规划 (Planning)

### 任务分解清单
- [ ] 制定实施计划
- [ ] 分解任务为可执行单元
- [ ] 估算工作量
- [ ] 确定优先级
- [ ] 规划测试策略

### 输出物
- 任务列表
- 优先级排序
- 里程碑定义

## 阶段 3: 架构 (Architecture)

### 任务分解清单
- [ ] 设计系统架构
- [ ] 定义模块边界
- [ ] 规划数据流
- [ ] 设计接口规范
- [ ] 评估性能影响

### 输出物
- 架构设计文档
- 模块依赖图
- 接口规范

## 阶段 4: 实施 (Implementation)

### 任务分解清单
- [ ] 按计划执行任务
- [ ] 编写代码和测试
- [ ] 代码审查
- [ ] 集成测试
- [ ] 文档更新

### 输出物
- 源代码
- 测试代码
- 技术文档

## 阶段 5: 部署 (Deployment)

### 任务分解清单
- [ ] 环境配置
- [ ] 部署验证
- [ ] 性能测试
- [ ] 监控配置
- [ ] 回滚计划

### 输出物
- 部署脚本
- 运维文档
- 监控配置

# 规模自适应指南

## 小型任务 (Bug Fix / 小功能)
- 跳过详细规划
- 直接实施
- 快速测试验证

## 中型任务 (功能模块)
- 简化分析
- 基本规划
- 架构审查
- 标准实施流程

## 大型任务 (系统级)
- 完整分析
- 详细规划
- 架构设计
- 分阶段实施
- 全面测试

## 企业级 (平台/多系统)
- 全流程执行
- 多轮迭代
- 跨团队协作
- 持续集成

# 代理角色定义

## PM 代理
- 职责: 需求分析、优先级排序、项目协调
- 输出: PRD、任务列表、里程碑

## 架构师代理
- 职责: 系统设计、技术选型、架构评审
- 输出: 架构文档、技术方案、接口规范

## 开发者代理
- 职责: 编码实现、单元测试、代码优化
- 输出: 源代码、测试代码、技术文档

## UX 代理
- 职责: 用户体验设计、交互设计
- 输出: 原型设计、交互规范

## QA 代理
- 职责: 测试策略、质量保证
- 输出: 测试计划、测试报告

## DevOps 代理
- 职责: 部署自动化、监控运维
- 输出: 部署脚本、监控配置

## 项目状态
- 阶段: 实施/优化

- ChromaDB (向量存储)

## 核心目标
1. 零硬编码 - 所有规则自主学习
2. 领域自适应 - 自动识别领域
3. 自我进化 - 持续学习成长

## 使用 BMAD 方法时的注意事项
- 优先使用 Evolution 层的自主学习能力
- 避免硬编码业务逻辑
- 所有新增功能需有测试覆盖
- 遵循统一逻辑表达层架构
