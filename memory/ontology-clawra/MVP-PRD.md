# 采购供应链选型助手 - MVP PRD

**版本**: v0.2  
**日期**: 2026-03-19  
**状态**: 草稿  
**聚焦场景**: 通用供应链采购决策辅助

---

## 一、产品定位

### 1.1 核心定位

**一句话描述**: 基于本体推理的可信采购决策助手，帮助采购人员从需求描述到推荐方案，提供全链路可解释的供应链决策支持。

### 1.2 差异化价值

| 维度 | 传统采购助手 | ontology-clawra |
|------|-------------|-----------------|
| 推荐依据 | 模糊匹配 | 4级置信度推理链 |
| 幻觉问题 | 无保障 | 每条建议带推理依据 |
| 学习能力 | 无 | 用户确认→自动抽取新实体 |
| 决策解释 | 无 | check→declare→confirm→label→reason |
| 供应链视角 | 单一比价 | 全链路风险/成本/交付评估 |

### 1.3 目标用户

- **Primary**: 中小企业采购工程师（无专业背景）
- **Secondary**: 采购主管（需要快速决策依据）
- **Tertiary**: 供应链管理人员（需要风险评估）

## 二、用户痛点（已验证）

### 2.1 核心痛点

| 痛点 | 描述 | 频次 | 成本影响 |
|------|------|------|---------|
| **选型困惑** | 不了解技术参数，面对大量规格无从下手 | 高 | 选错型号→返工/停机 |
| **信息不对称** | 供应商信息分散，难以横向对比 | 高 | 报价虚高 |
| **决策无据** | 选型依据无法说服上级/审计 | 中 | 决策反复 |
| **供应链风险** | 供应商单一依赖、交付风险、库存积压 | 高 | 断供/呆滞成本 |
| **知识断层** | 老员工离职，新人上手慢 | 中 | 培训成本高 |

### 2.2 场景举例

> **用户故事1**: 采购员小王需要采购一批工业阀门，面对10家供应商的20种型号，完全不知道参数差异代表的实际意义，选型后被工程师退货。

> **用户故事2**: 采购主管老李需要向领导汇报选型依据，但除了价格和产品手册，无法提供有理有业的决策说明。

> **用户故事3**: 供应链经理老张发现某个关键物料只有单一供应商，想评估替代供应商风险，但缺乏系统化的评估方法。

## 三、MVP功能设计

### 3.1 功能架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户交互层                            │
│  [需求描述] → [方案推荐] → [供应商评估] → [决策建议]      │
│  [库存咨询] → [风险识别] → [成本分析]                    │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                    推理引擎层                            │
│  [本体匹配] → [规则推理] → [置信度计算] → [可解释输出]    │
│  [供应链风险评估] → [成本优化推理]                        │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                    知识库层                              │
│  [产品本体库] → [供应商库] → [价格行情] → [案例库]        │
│  [供应链风险库] → [库存策略库]                           │
└─────────────────────────────────────────────────────────┘
```

### 3.2 MVP核心功能

#### 功能1: 智能选型推荐（采购需求→推荐方案）

**输入**: 用户描述需求（自然语言或参数）  
**输出**: 推荐方案 + 推理依据 + 置信度

**示例交互**:
```
用户: "我们车间需要采购一批气动阀门，介质是天然气，口径DN50，压力1.0MPa，要求防爆"
    
ontology-clawra:
┌─────────────────────────────────────────────────────────┐
│ 推荐: XXX-DN50 气动球阀                                 │
├─────────────────────────────────────────────────────────┤
│ 推理依据:                                               │
│ 1. [规则-001] 口径DN50 → 匹配DN50系列 ✓                │
│ 2. [规则-002] 压力1.0MPa → 适用PN16等级 ✓              │
│ 3. [规则-003] 天然气介质 → 需防爆认证 ✓                │
│ 4. [规则-004] 匹配度评分: 92/100                        │
├─────────────────────────────────────────────────────────┤
│ 置信度: ●●●●○ 高 (85%)                                  │
│ 备选: YYY-DN50 (75%)                                   │
└─────────────────────────────────────────────────────────┘
```

#### 功能2: 供应商评估（供应链视角）

**输入**: 供应商名称/物料类别  
**输出**: 风险评分 + 详细说明 + 改进建议

**评估维度**:
- **资质合规性**: 营业执照、认证证书、行业资质
- **产品质量**: 抽检历史、投诉记录、品牌口碑
- **交付能力**: 交期准时率、历史履约、备货能力
- **价格竞争力**: 市场均价对比、议价空间
- **供应链风险**: 单一依赖、地理风险、替代供应商可获得性

#### 功能3: 采购决策建议

**输入**: 候选方案列表  
**输出**: 优先级排序 + 决策依据 + 风险提示

**示例**:
```
方案A: 供应商A - ¥50,000 - 交期15天 - 质保2年 - 品牌头部
方案B: 供应商B - ¥45,000 - 交期20天 - 质保1年 - 品牌一般

推荐: 方案A
依据: [规则-010] 质保期每增加1年，ROI提升12%
      [规则-011] 交期减少5天，库存成本降低¥800
      [规则-012] 头部品牌降低质量风险评分15%
综合评分: A=85, B=72
```

#### 功能4: 供应链风险识别

**输入**: 物料/供应商列表  
**输出**: 风险点清单 + 应对建议

**风险类型**:
- 单一供应商依赖
- 交付周期波动
- 价格波动趋势
- 质量风险聚集
- 地理/地缘风险

### 4.1 主流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  需求   │───→│  推荐   │───→│  评估   │───→│  决策   │
│  输入   │    │  方案   │    │  供应商 │    │  确认   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │
     ▼              ▼              ▼              ▼
  [参数/        [AI推理      [风险/成本    [导出
   场景]        推荐]        多维评估]      依据]

┌─────────┐    ┌─────────┐
│  风险   │───→│  优化   │
│  识别   │    │  建议   │
└─────────┘    └─────────┘
     │              │
     ▼              ▼
  [供应链      [库存/交付
   风险]       优化建议]
```

### 4.2 用户故事（≥3个）

#### 用户故事1: 选型不踩坑

**角色**: 采购工程师小张  
**场景**: 车间改造需要采购5台防爆控制柜  
**痛点**: 参数复杂，不确定哪些是必须的，哪些是可选的  
**期望**: 输入基本需求后，AI能告诉我为什么选这个方案

**流程**:
1. 小张输入: "需要5台防爆控制柜，用于燃气车间，II区防爆"
2. 系统返回推荐方案及推理链
3. 小张追问: "这个和XXX方案有什么区别?"
4. 系统对比差异点及影响
5. 小张确认选择，导出选型报告

**验收标准**:
- [ ] 输入需求后5秒内返回推荐
- [ ] 每条推荐附带至少3条推理依据
- [ ] 支持追问式对话

#### 用户故事2: 供应商评估

**角色**: 采购主管老李  
**场景**: 供应商年度审核，需要评估20家供应商  
**痛点**: 人工评估耗时，且缺乏统一标准  
**期望**: 输入供应商名称，自动生成评估报告

**流程**:
1. 老李导入供应商名单
2. 系统自动抓取公开信息+历史交易数据
3. 生成风险评估报告（可视化）
4. 老李复核，标注关键风险点
5. 导出评估结果

**验收标准**:
- [ ] 单次评估≤10家供应商
- [ ] 每家供应商包含≥5个评估维度
- [ ] 支持导出PDF/Excel报告

#### 用户故事3: 决策依据

**角色**: 采购员小王  
**场景**: 领导要求说明为什么选这家供应商  
**痛点**: 只能说出价格低，无法提供专业依据  
**期望**: AI帮我生成有理有据的决策说明

**流程**:
1. 小王选择候选供应商A和B
2. 系统生成对比分析报告
3. 报告包含: 价格、技术、交付、风险多维度对比
4. 每项对比附带推理依据和置信度
5. 小王将此报告提交领导

**验收标准**:
- [ ] 对比维度≥6个
- [ ] 每维度提供量化依据
- [ ] 支持决策理由生成（自然语言）

#### 用户故事4: 供应链风险识别

**角色**: 供应链经理老张  
**场景**: 年度供应链审计，发现关键物料依赖单一供应商  
**痛点**: 不知道如何系统评估风险，也没有替代方案  
**期望**: AI帮我识别供应链风险并给出优化建议

**流程**:
1. 老张导入关键物料清单
2. 系统分析供应商依赖情况
3. 识别单一供应商风险点
4. 推荐替代供应商候选
5. 给出库存/交付优化建议

**验收标准**:
- [ ] 识别关键风险点≥3个
- [ ] 每风险点附带应对建议
- [ ] 推荐替代供应商≥2家

## 五、成功指标

### 5.1 核心指标

| 指标 | 定义 | MVP目标 | 测量方式 |
|------|------|---------|---------|
| **采纳率** | 用户采纳AI推荐的比例 | ≥60% | 埋点统计 |
| **满意度** | 用户对推理依据的认可度 | ≥4.0/5 | 事后评分 |
| **效率提升** | 选型时间减少 | ≥50% | 对比测试 |
| **置信度达标率** | 高置信度(>80%)推荐占比 | ≥70% | 系统统计 |

### 5.2 辅助指标

| 指标 | 定义 | MVP目标 |
|------|------|---------|
| **对话轮次** | 平均咨询轮次 | ≤3轮 |
| **追问率** | 用户追问比例 | ≤40% |
| **学习触发** | 主动学习触发次数 | 记录 |
| **错误反馈** | 用户纠正AI的次数 | 记录 |

### 5.3 验收条件

**MVP发布标准**:
- [ ] 功能1（选型推荐）可演示
- [ ] 推理依据可读、可理解

- [ ] 响应时间<5秒

## 六、技术约束

### 6.1 聚焦领域

**MVP选择**: 通用工业品/设备选型  
**理由**:
1. 参数化程度高，适合本体推理
2. 供应链复杂度高，可解释性有价值
3. 竞品覆盖少，差异化明显
4. 可扩展至多品类

### 6.2 暂不覆盖

- [ ] 全品类采购
- [ ] 合同风险识别
- [ ] 实时价格监控
- [ ] ERP/MRP集成

### 6.3 领域扩展路径

```
MVP (v0.1): 单一垂直领域 (如: 工业阀门)
    ↓
v0.2: 扩展至3-5个相关品类 (阀门、管道、仪表)
    ↓
v0.3: 通用工业品/设备 (覆盖80%常见品类)
    ↓
v1.0: 全品类采购支持
```
