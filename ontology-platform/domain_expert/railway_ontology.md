# 铁路领域本体设计

## 概述

本文档定义铁路领域的核心本体模型，涵盖铁路运输的基础设施、车辆设备、运营管理、维修保养等核心领域，用于指导知识库构建和智能化应用。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 铁路线路 | Railway Line | 铁路运输的物理通道 | 线路编码、名称、起点、终点、里程、等级 |
| 车站 | Station | 旅客上下车和货物装卸的设施 | 车站编码、名称、位置、等级、股道数 |
| 轨道 | Track | 供列车行驶的轨道结构 | 轨道ID、线路ID、里程标、轨道类型 |
| 信号机 | Signal | 指挥列车运行的信号设备 | 信号机编号、位置、类型、显示状态 |
| 机车 | Locomotive | 牵引列车的动力机车 | 机车编号、型号、功率、牵引力 |
| 动车组 | EMU Train | 动力分散式列车组 | 车组号、车型、定员、速度等级 |
| 客车 | Passenger Car | 运送旅客的车厢 | 车厢编号、车型、座位数、舒适度等级 |
| 货车 | Freight Car | 运输货物的车辆 | 车皮编号、载重、车型、货类型 |
| 列车 | Train | 机车或动车组与车厢组成的运行单元 | 车次、编组、起始站、终到站、时刻 |
| 时刻表 | Timetable | 列车运行时刻安排 | 时刻表ID、版本、生效日期、编制规则 |
| 调度指令 | Dispatching Order | 调度员发布的运行指挥命令 | 指令编号、时间、发令人、指令内容 |
| 票务系统 | Ticketing System | 旅客票务发售和退改签系统 | 系统ID、票价规则、余票信息 |
| 乘务组 | Crew | 列车值乘人员 | 班组编号、成员、值乘区间、值乘时间 |
| 维修工区 | Maintenance Depot | 铁路设备维修保养的作业单元 | 工区编码、管辖范围、设备数量、人员配置 |
| 巡检 | Inspection | 设备检查维护作业 | 巡检ID、路线、设备状态、发现问题 |

## 二、规律 (Laws)

### 2.1 列车追踪间隔原理

**定义**: 同一线路上前后两列列车必须保持的最小安全间隔距离。

**核心原则**:
```
追踪间隔 ≥ 制动距离 + 安全余量
```

**影响因素**:
- 列车运行速度
- 制动性能
- 线路坡度
- 天气条件 (雨雪冰冻)
- 信号系统类型

**典型间隔**:
- 高速铁路: 3-5分钟
- 普速铁路: 6-10分钟
- 城市轨道: 2-3分钟

### 2.2 铁路通过能力计算

**定义**: 铁路线路在单位时间内能够通过的最大列车数量。

**计算公式**:
```
通过能力 (列/日) = 1440 / (追踪间隔 + 停站时间 + 调整余量)
```

**限制因素**:
- 区间通过能力
- 车站到发能力
- 咽喉通过能力
- 整备能力

### 2.3 旅客列车票价率

**定义**: 旅客乘坐铁路列车旅行时，票价与里程的比率关系。

**基本票价率** (以公布票价为准):
```
基础票价 = 票价率 × 里程
```

**票价率等级**:
| 座位类型 | 票价率 (元/人公里) |
|----------|-------------------|
| 硬座 | 0.05861 |
| 软座 | 0.11750 |
| 硬卧 | 0.10220 |
| 软卧 | 0.17650 |
| 商务座 | 0.37460 |

**浮动规则**:
- 旺季 (节假日): 上浮最高50%
- 淡季: 下浮最高20%
- 弹窗车票: 实时浮动

### 2.4 列车重量与速度匹配规律

**定义**: 列车牵引力与运行速度之间的匹配关系。

**基本原理**:
```
P = F × v / 3600
```
其中: P=功率(kW), F=牵引力(N), v=速度(km/h)

**常见配置**:
| 车型 | 牵引功率 (kW) | 最高速度 (km/h) | 适用场景 |
|------|---------------|-----------------|----------|
| 和谐号CRH380A | 15600 | 380 | 高速客运 |
| 和谐号CRH2 | 7200 | 250 | 城际客运 |
| 复兴号CR400AF | 10140 | 350 | 高速客运 |
| HXD1型电力机车 | 9600 | 120 | 货运重载 |

### 2.5 铁路维修周期规律

**定义**: 铁路设备和设施的预防性维修时间间隔。

**主要维修周期**:
| 设备类型 | 维修周期 | 维修内容 |
|----------|----------|----------|
| 钢轨 | 伤损检查: 月度<br>打磨: 100-200Mt通过量 | 伤损检测、砂轮打磨 |
| 道岔 | 周检查、月保养、季度检修 | 动作试验、润滑、调整 |
| 接触网 | 3-5年大修、季度检测 | 悬挂调整、绝缘子清洁 |
| 信号设备 | 日监测、月检修、季测试 | 电气特性测试、模拟试验 |

### 2.6 列车运行能耗规律

**定义**: 列车运行过程中的能源消耗特性。

**能耗计算**:
```
能耗 = 基础能耗 + 附加能耗
基础能耗 ∝ 列车重量 × 运行里程
附加能耗 = 坡道能耗 + 启动能耗
```

**典型能耗指标**:
| 列车类型 | 单位能耗 (kWh/万人公里) |
|----------|------------------------|
| 高速动车组 | 35-50 |
| 城际动车组 | 25-35 |
| 普速客车 | 15-25 |
| 货运列车 | 8-12 (吨公里) |

## 三、业务规则 (Rules)

### 3.1 列车开行条件

| 条件类型 | 要求 |
|----------|------|
| 线路条件 | 线路通过验收，具备开行条件 |
| 车辆条件 | 车辆技术状态良好，在检修周期内 |
| 人员条件 | 司机、乘务员已培训合格并取得资质 |
| 安全保障 | 信号设备正常、备品齐全 |
| 文件条件 | 时刻表已公布、编组已批准 |

### 3.2 调度命令发布规则

**发布权限**:
| 调度级别 | 可发布命令类型 |
|----------|----------------|
| 铁路总公司调度 | 全路性调度命令、跨局命令 |
| 铁路局调度 | 局管内运输调整命令 |
| 车站值班员 | 站内调车作业命令 |

**命令要素**:
- 命令编号 (按日编号)
- 发布时间 (年-月-日-时-分)
- 发令人职务和姓名
- 受令人职务和姓名
- 命令内容 (必须明确具体)
- 执行时间

### 3.3 车站接发列车作业标准

**接车作业流程**:
```
1. 办理预告 → 2. 确认接车线 → 3. 开放信号 → 4. 接车通报 → 5. 列车到达 → 6. 停车防溜
```

**发车作业流程**:
```
1. 准备发车 → 2. 确认发车条件 → 3. 开放信号 → 4. 准备进路 → 5. 指示发车 → 6. 列车出发
```

### 3.4 铁路限界标准

**建筑限界**: 铁路两侧建筑物不得侵入的空间范围

**机车车辆限界**:
| 限界类型 | 宽度 (mm) | 高度 (mm) |
|----------|-----------|-----------|
| 最高等级 | 1700 | 4800 |
| 一般铁路 | 1700 | 4600 |
| 城市轨道 | 1400 | 3800 |

### 3.5 乘务组值乘规则

**值乘时间限制**:
- 单司机值乘: 最长4小时
- 双司机值乘: 最长6小时
- 换班间隔: 不少于30分钟

**值乘前准备**:
- 按时出勤报到
- 抄写运行揭示
- 领取行车备品
- 进行酒精测试
- 召开出乘小组会

### 3.6 铁路交通事故分类

| 事故等级 | 死亡人数 | 或重伤人数 | 或直接经济损失 |
|----------|----------|------------|----------------|
| 特别重大事故 | ≥30人 | ≥100人 | ≥1亿元 |
| 重大事故 | 10-29人 | 50-99人 | 5000万-1亿元 |
| 较大事故 | 3-9人 | 10-49人 | 1000万-5000万元 |
| 一般事故 | 1-2人 | 1-9人 | 200万-1000万元 |

### 3.7 铁路工程质量验收标准

**单位工程验收** → **分部工程验收** → **静态验收** → **动态验收** → **初步验收** → **安全评估** → **正式验收**

**关键验收指标**:
| 验收项目 | 合格率要求 |
|----------|------------|
| 轨道几何尺寸 | 100%合格 |
| 接触网参数 | ≥95%达标 |
| 信号设备 | 100%功能正常 |
| 通信质量 | ≥98%可用 |

### 3.8 列车运行图编制规则

**基本要素**:
- 列车区间运行时间
- 车站停站时间
- 列车会让方案
- 车站到发线运用
- 机车车辆运用

**编制原则**:
1. 满足客流需求
2. 充分利用通过能力
3. 保证行车安全
4. 经济合理使用机车车辆
5. 方便旅客出行

## 五、应用场景

1. **智能问答**: 基于本体知识回答铁路线路、列车时刻、票价规则等常见问题
2. **调度辅助**: 根据实时数据提供调度建议、冲突检测、延误预警
3. **设备管理**: 基于维修周期规律进行预防性维修计划和状态监测
4. **安全管理**: 通过本体关系识别安全隐患、追溯事故原因
5. **客流分析**: 结合票务数据预测客流趋势、优化运力配置
6. **应急指挥**: 突发事件时快速定位受影响区段、调度救援资源
7. **知识推理**: 通过本体关系发现潜在风险、规划新线建设
