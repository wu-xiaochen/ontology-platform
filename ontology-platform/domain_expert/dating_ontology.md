# 约会领域本体设计

## 概述

本文档定义约会（Dating）领域的核心本体模型，用于指导婚恋平台、智能匹配系统、约会助手等应用的知识库构建和智能化服务。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 用户 | User | 约会平台的注册用户 | 用户ID、昵称、年龄、性别、地理位置、兴趣爱好、活跃度 |
| 个人资料 | Profile | 用户的详细资料信息 | 头像、简介、职业、学历、身高、收入、婚姻状况、认证状态 |

## 二、规律 (Laws)

### 2.1 匹配算法核心规律

**相似性匹配定律**

| 匹配维度 | 权重 | 规律描述 | 最佳范围 |
|----------|------|----------|----------|
| 地理位置 | 15% | 距离越近，匹配成功率高 | 5-20km |
| 年龄匹配 | 10% | 年龄差在3-5岁内匹配度最高 | 差值≤5岁 |
| 兴趣爱好 | 25% | 共同兴趣数量与满意度正相关 | ≥3个共同兴趣 |
| 价值观 | 30% | 核心价值观契合决定长期关系 | 核心价值匹配度≥80% |
| 生活方式 | 20% | 生活节奏相似度影响相处舒适度 | 作息、饮食习惯相似 |

**匹配度计算公式**:
```
匹配度 = Σ(维度权重 × 维度相似度) × 活跃度系数 × 互惠系数

其中:
- 活跃度系数 = min(1.2, log(用户近30天活跃天数 + 1))
- 互惠系数 = 0.8(单方面喜欢) 或 1.0(双向喜欢)
```

**漏斗模型**:
```
潜在匹配池 → 筛选匹配 → 匹配成功 → 首次聊天 → 首次约会 → 确定关系
转化率: 100% → 40% → 60% → 35% → 25% → 15%
```

### 2.2 约会成功率规律

**初次约会黄金法则**

| 因素 | 影响 | 最佳实践 | 成功率影响 |
|------|------|----------|-----------|
| 约会时长 | 成功率与时长呈倒U型 | 1.5-2.5小时 | 过长<2h +8%, >3h -5% |
| 约会时间 | 周末下午 > 工作日晚间 | 周六下午2-5点 | 较之工作日晚 +12% |
| 场所选择 | 半公开场所成功率高 | 咖啡厅、餐厅、公园 | 较之私密场所 +15% |
| 主动程度 | 双方主动度平衡 | 轮流主动邀约 | 一方单向主动 -10% |
| 话题深度 | 中等深度最适宜 | 避免过私密或表面 | 适度深度话题 +8% |

**约会阶段成功率统计**:
```
第一次约会成功率: 35-45%
第二次约会邀请率: 60-70%
第三次约会确定关系率: 40-50%
五次约会后稳定交往率: 70-80%
```

**约会转化漏斗**:
```
匹配 → 首次聊天: 90%
首次聊天 → 交换联系方式: 70%
交换联系方式 → 首次约会: 50%
首次约会 → 第二次约会: 65%
第二次约会 → 确定关系: 45%
```

### 2.3 消息互动规律

**回复率时间规律**

| 消息发送时间 | 平均回复率 | 最佳时段特征 | 建议发送窗口 |
|--------------|------------|--------------|--------------|
| 早上7-9点 | 45% | 通勤时间，快速回复 | 7:30-8:30 |
| 上午9-12点 | 35% | 工作时间，回复慢 | 不宜发送重要消息 |
| 下午12-14点 | 50% | 午休放松时段 | 12:30-13:30 |
| 下午14-18点 | 40% | 工作间隙 | 16:00-17:00较佳 |
| 晚上18-21点 | 55% | 黄金时段，最高活跃 | 19:00-20:30 |
| 晚上21-23点 | 48% | 睡前浏览，放松 | 21:30-22:30 |
| 深夜23点后 | 30% | 休息时间，不推荐 | 避免发送 |

**消息长度与回复率**:
```
过短 (<5字): 回复率 35% (显得敷衍)
适中 (10-30字): 回复率 65% (最佳区间)
过长 (>100字): 回复率 40% (阅读负担)

含图片消息回复率: +15%
问句回复率比陈述句: +20%
```

**消息互动黄金法则**:
- 首条消息24小时内回复率最高，超过72小时回复率下降50%
- 连续三次未回复建议暂停3-7天
- 回复间隔时间逐渐缩短是积极信号
- 消息轮次达到50轮以上，首次约会的转化率提升40%

### 2.4 关系发展规律

**关系发展时间线**

| 阶段 | 平均时长 | 关键里程碑 | 成功率 |
|------|----------|------------|--------|
| 匹配-首次聊天 | 24-72小时 | 首日互动很重要 | 72小时内互动成功率+25% |
| 聊天-交换联系方式 | 3-7天 | 微信/电话交换 | 未交换则匹配流失率+30% |
| 联系方式-首次约会 | 7-14天 | 首次见面 | 超过21天首次约会概率-40% |
| 首次约会-第二次 | 2-5天 | 反馈及时性 | 3天内邀约成功率+20% |
| 多次约会-确定关系 | 2-6周 | 表白/确认 | 超过8周不确定关系概率+35% |
| 恋爱-同居 | 6-18个月 | 考虑现实因素 | - |

**关系进展标志信号**:
- ✅ 愿意主动发消息：兴趣指标 (+25%)
- ✅ 消息内容逐渐深入：信任建立 (+20%)
- ✅ 同意快速见面：意愿强烈 (+30%)
- ✅ 愿意分享日常：亲密感 (+15%)
- ✅ 主动肢体接触：关系升级信号 (+35%)

**关系降级预警信号**:
- ❌ 回复间隔超过48小时
- ❌ 回复内容简短敷衍 (≤5字)
- ❌ 连续3次拒绝约会邀请
- ❌ 不再主动发起对话
- ❌ 社交平台取关或屏蔽

### 2.5 用户行为规律

**活跃度与匹配成功率关系**

| 每日活跃时长 | 周均匹配数 | 首次约会转化率 | 7日留存率 |
|--------------|------------|----------------|-----------|
| <30分钟 | 1.2 | 8% | 35% |
| 30-60分钟 | 3.5 | 15% | 52% |
| 60-120分钟 | 8.7 | 28% | 68% |
| >120分钟 | 12.3 | 32% | 72% |

**付费用户行为特征**:
- 匹配成功率: +40%
- 首次约会转化率: +35%
- 平均消息轮次: +25轮
- 关系确定周期: 缩短30%

## 三、业务规则 (Rules)

### 3.1 用户匹配规则

**匹配准入条件**

| 条件类型 | 具体要求 | 说明 |
|----------|----------|------|
| 基础条件 | 双方均已完成资料认证 | 认证通过率≥85% |
| 年龄限制 | 男方22-60岁，女方20-55岁 | 超出范围需特殊审批 |
| 地区限制 | 同城或邻近城市(≤100km) | 同城优先 |
| 状态限制 | 双方均为单身状态 | 需定期更新 |
| 互惠规则 | 双方至少一方对另一方有好感 | 单向喜欢不计入匹配 |

**匹配推荐优先级**

| 优先级 | 推荐规则 | 候选人数 | 展示位置 |
|--------|----------|----------|----------|
| P0 | 高度匹配(匹配度>85%) + 活跃用户 | 5-10人 | 优先展示 |
| P1 | 高匹配度(70-85%) + 最近7天登录 | 10-20人 | 次优先 |
| P2 | 中等匹配度(50-70%) + 共同兴趣≥5个 | 20-50人 | 补充推荐 |
| P3 | 扩展推荐(匹配度<50%) + 新用户优先 | 0-30人 | 探索发现 |

**匹配冷却规则**:
- 24小时内最多推荐50人
- 对同一用户重复推荐间隔≥7天
- 已拒绝用户30天内不再推荐

### 3.2 消息互动规则

**消息发送规则**

| 规则类型 | 具体内容 | 违规处理 |
|----------|----------|----------|
| 匹配后首条消息 | 24小时内发送最佳，超过7天未发默认放弃 | 自动取消匹配，释放双方额度 |
| 消息频率 | 首日1-3条为宜，同时间段间隔≥15分钟 | 超过5条/小时触发限流 |
| 敏感词过滤 | 自动过滤广告、诈骗、违规内容 | 警告→禁言→封号 |
| 敏感话题 | 初次聊天避免收入、婚史、前任等话题 | 触发警告，多次违规限流 |
| 互动保护 | 连续3天未回复可提醒一次 | 超过5天未回复自动结束对话 |

**消息内容规范**

| 推荐话题 | 避免话题 | 风险等级 |
|----------|----------|----------|
| 兴趣爱好 | 收入存款 | 高 |
| 旅行经历 | 前任情况 | 高 |
| 美食探索 | 身体状况 | 中 |
| 电影音乐 | 政治宗教 | 高 |
| 职业趣事 | 相亲动机 | 低 |
| 童年往事 | 催婚压力 | 中 |

**消息撤回规则**:
- 24小时内可撤回
- 已读消息撤回需对方同意
- 撤回后不可编辑，只能重新发送

### 3.3 约会安排规则

**首次约会规范**

| 要素 | 建议规范 | 原因 |
|------|----------|------|
| 时间 | 周末下午2-5点为宜 | 精神状态好，有后续时间 |
| 时长 | 1.5-2小时 | 倒U型曲线，适中最佳 |
| 场所 | 公共场所、交通便利 | 安全考虑，便于离开 |
| 消费 | 建议AA制或轮流请客 | 避免尴尬，体现平等 |
| 安全 | 告知亲友行程、保持警惕 | 防风险 |
| 着装 | 干净整洁，符合场合 | 第一印象 |

**约会场所选择规则**

| 关系阶段 | 推荐场所类型 | 不推荐 | 成功率对比 |
|----------|--------------|--------|------------|
| 初次约会 | 咖啡厅、书店、公园 | 密室、电影院 | 基准100% |
| 第二次约会 | 餐厅、展览、动物园 | 家中、私人影院 | +15% |
| 熟悉后 | 徒步、做饭、周边游 | - | +25% |
| 确定关系 | 正式餐厅、浪漫场所 | - | +35% |

**约会确认流程**:
```
邀约发起 → 场所确认 → 时间协商 → 正式确认 → 提醒通知(提前1天+2小时) → 发生约会 → 评价反馈
```

**取消规则**:
- 提前24小时取消：无惩罚
- 2-24小时取消：扣除5积分（可申诉）
- 2小时内取消：扣除15积分，影响信用分
- 无故缺席：扣除30积分，临时封禁

### 3.4 关系进展规则

**升级关系触发条件**

| 关系升级 | 触发条件 | 确认方式 | 系统处理 |
|----------|----------|----------|----------|
| 聊天→约会 | 聊天超过50条 + 3天以上 + 互有好感 | 主动邀约 + 同意 | 升级为"已约会"状态 |
| 约会→暧昧 | 约会2次以上 + 互动积极(Positive≥80%) | 肢体接触 + 语言暗示 | 系统标记"暧昧期" |
| 暧昧→表白 | 相处超过1个月 + 明确暗示 | 语言确认"做我男/女朋友" | 正式恢复"恋爱中" |
| 交往→同居 | 交往6个月以上 + 共同意愿 | 双方同意并设置状态 | - |

**关系降级/终止规则**

| 情况 | 处理方式 | 系统动作 |
|------|----------|----------|
| 单方面冷淡 | 3次主动无果后建议暂停 | 发送提醒，30天后自动降级 |
| 明确拒绝 | 尊重对方，停止追求 | 立即终止匹配，双方解绑 |
| 违反底线 | 立即终止并举报 | 冻结账户，人工审核 |
| 长时间失联 | 7天无回应可标记放弃 | 自动解绑，释放匹配额度 |

**状态机模型**:
```
(陌生人)→(已匹配)→(聊天中)→(约会中)→(暧昧期)→(恋爱中)
   ↓          ↓          ↓          ↓          ↓
 (放弃)    (未聊天)   (无约会)    (冷处理)   (分手)
```

### 3.5 用户行为规范

**用户行为准则**

| 规范类型 | 具体要求 | 监控指标 |
|----------|----------|----------|
| 真实性 | 资料信息需真实可靠，禁止虚构造假 | 举报率、被拒绝率 |
| 尊重性 | 禁止骚扰、辱骂、威胁等行为 | 敏感词、被举报次数 |
| 诚信性 | 禁止骗婚、骗财、传销等行为 | 金钱相关话题、转账警告 |
| 隐私性 | 未经允许禁止泄露他人隐私 | 个人信息分享、外链 |
| 安全性 | 禁止线下交易、诱导投资等行为 | 敏感词、行为模式 |

**违规行为处罚**

| 违规级别 | 行为描述 | 处罚措施 | 申诉时效 |
|----------|----------|----------|----------|
| 轻微 | 资料不完整、频繁换头像 | 提示完善，限流3天 | 3天 |
| 一般 | 频繁骚扰(3次/天)、虚假信息 | 限流7-30天 | 7天 |
| 严重 | 诈骗、违法内容、言语攻击 | 永久封号 | 14天 |
| 极其严重 | 违法犯罪、人身威胁 | 立即封号 + 移交司法 | 不可申诉 |

**信用分体系**:
- 初始分：100分
- 正常约会：+1分/次
- 取消约会：-5分(提前24h) / -15分(<24h)
- 被举报核实：-30分
- 满分150，低于60分限制功能
- 每月恢复5分，最低不低于60分

### 3.6 数据隐私规则

**个人信息保护**:
- 手机号、微信号仅双方匹配后可见
- 详细地址仅约会确认后临时可见
- 聊天内容端到端加密，不存储明文
- 用户可随时导出/删除自己的数据

** GDPR合规**:
- 明确告知数据用途
- 用户可以随时注销账户
- 注销后30天内完全删除数据
- 未成年人禁止注册

## 五、应用场景

### 5.1 智能匹配系统

**精准推荐**: 基于用户画像和偏好进行智能匹配，使用协同过滤+内容推荐混合算法

**匹配度预测**: 预测匹配成功的可能性，使用机器学习模型
- 特征: 用户画像、历史行为、社交图谱
- 模型: GBDT + 神经网络
- AUC目标: >0.85

**最优推荐排序**: 根据多维度因素排序推荐列表
- 综合得分 = 匹配度 × 0.6 + 活跃度 × 0.2 + 新鲜度 × 0.2
- 新鲜度衰减因子: 每过24小时 × 0.9

**冷启动处理**:
- 新用户: 基于注册信息和相似用户推荐
- 行为数据积累≥20次互动后切换为个性化推荐

### 5.2 约会助手

**开场白建议**: 根据对方资料生成个性化开场白
- 提取对方兴趣标签
- 生成3个候选开场白
- A/B测试选择效果最好的模板

**话题推荐**: 根据共同兴趣推荐聊天话题
- 识别双方共同兴趣
- 生成话题树(浅→深)
- 避免敏感话题自动过滤

**约会策划**: 智能推荐约会场所和活动
```
输入: 用户A偏好 + 用户B偏好 + 关系阶段 + 预算
输出: 3个推荐方案(场所+活动+餐饮)
打分: 匹配度 × 环境适合度 × 性价比
```

**时机提醒**: 提醒最佳发消息时间、约会邀请时机
- 基于对方活跃度数据
- 预测最佳回复窗口
- 约会邀约提前2-3天发送

### 5.3 关系顾问

**关系诊断**: 分析关系状态和发展趋势
- 聊天热情度分析(消息频率、长度、回复速度)
- 约会进度评估(是否符合时间线)
- 风险信号识别(冷淡、拒绝)

**问题预警**: 识别潜在问题和风险
- 连续3天未主动发消息 → 预警冷淡
- 回复间隔持续延长 → 预警兴趣下降
- 拒绝约会超过2次 → 预警破裂

**改进建议**: 提供关系改善建议
- 根据当前阶段推荐行动
- 话题建议、约会创意
- 沟通技巧提示

**情感分析**: 分析双方互动中的情感变化
- 文本情感分析(积极/消极/中性)
- 情感趋势曲线
- 识别重要情感转折点

### 5.4 安全防护

**身份认证**: 辅助验证用户真实身份
- 实名认证(身份证OCR)
- 人脸识别活体检测
- 学历/职业认证(学信网/企业邮箱)

**风险识别**: 识别诈骗倾向和不良用户
- 敏感词检测(金钱、投资、赌博)
- 行为模式分析(频繁邀约、索要钱财)
- 黑名单库比对

**安全提醒**: 约会安全注意事项提醒
- 首次约会安全提示
- 场所安全评分
- 紧急联系方式

**紧急求助**: 突发情况下的快速求助机制
- 一键发送位置给紧急联系人
- 平台客服快速响应
- 报警联动(需用户授权)

### 5.5 数据决策支持

**平台运营看板**:
- 每日新增用户、匹配数、约会数
- 转化漏斗(匹配→约会→确定关系)
- 留存曲线(7日、30日、90日)

**用户画像分析**:
- 用户分层(高价值/潜力/流失)
- 行为聚类分析
- 偏好洞察报告

**A/B测试平台**:
- 推荐算法迭代
- UI/UX优化
- 运营策略验证

| 匹配成功率 | 成功匹配数 / 主动匹配请求数 | ≥40% | 每日 |
| 首次约会转化率 | 首次约会数 / 匹配成功数 | ≥30% | 每周 |
| 关系确认率 | 确认关系数 / 首次约会数 | ≥25% | 每月 |
| 用户7日留存 | 7日内活跃用户数 / 新用户数 | ≥45% | 每日 |
| 付费转化率 | 付费用户数 / 总用户数 | ≥8% | 每周 |
| DAU/MAU | 日活跃用户数 / 月活跃用户数 | ≥0.3 | 每日 |
| 用户生命周期价值(LTV) | 平均用户付费总额 | ≥¥200 | 每月 |
| CAC/LTV | 获客成本 / 用户生命周期价值 | ≤1:3 | 每月 |

### 6.2 用户体验指标

| 指标名称 | 计算方式 | 目标值 | 优化方向 |

| 约会满意度 | 好评约会数 / 总约会数 | ≥80% | 场所质量提升 |
| 关系持续率 | 3个月后仍交往 / 确认关系数 | ≥50% | 关系辅导服务 |

### 6.3 匹配质量指标

| 指标名称 | 计算方式 | 目标范围 | 说明 |
|----------|----------|----------|------|
| 平均匹配度分数 | 所有匹配对的匹配度均值 | 70-85分 | 过高→推荐过窄，过低→质量差 |
| 匹配多样性 | 推荐用户的特征分布方差 | 较高 | 避免信息茧房 |
| 长尾用户覆盖率 | 匹配到用户的百分比 | ≥90% | 确保新用户和低活跃用户也能匹配 |
| 兴趣重合度 | 双方共同兴趣数均值 | ≥3个 | 共同兴趣是成功的关键因素 |
| 地理位置匹配率 | 同城匹配占比 | ≥60% | 异地匹配成功率显著降低 |

### 6.4 约会质量指标
