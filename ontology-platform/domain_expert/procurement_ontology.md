# 采购供应链领域本体设计

## 概述

本文档定义采购供应链领域的核心本体模型，用于指导知识库构建和智能化应用。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 供应商 | Supplier | 提供物料或服务的企业主体 | 供应商编码、名称、等级、资质、联系方式 |
| 采购商 | Buyer | 采购商品或服务的企业主体 | 企业编码、名称、行业、规模 |
| 物料 | Material | 采购或供应的物品 | 物料编码、名称、规格、类别、单位 |
| 采购订单 | Purchase Order | 采购活动的正式书面确认 | 订单编号、日期、数量、单价、金额 |
| 合同 | Contract | 采购双方达成的法律协议 | 合同编号、类型、有效期、条款 |
| 仓库 | Warehouse | 存储物料的设施 | 仓库编码、位置、容量、类型 |
| 库存 | Inventory | 仓库中物料的实际数量 | 物料ID、仓库ID、数量、批次 |
| 物流 | Logistics | 物料从供应商到采购商的运输过程 | 物流单号、运输方式、状态、轨迹 |

### 1.2 关系定义

```
供应商 ⊚ 供应 [物料] (多对多)
供应商 ⊚ 签订 [合同] (一对多)
采购商 ⊚ 下达 [采购订单] (一对多)
采购订单 ⊚ 关联 [物料] (多对一)
采购订单 ⊚ 触发 [物流] (一对多)
仓库 ⊚ 存储 [库存] (一对多)
物料 ⊚ 属于 [物料类别] (多对一)
```

### 1.3 物料类别体系

```
物料
├── 原材料 (Raw Material)
│   ├── 金属材料
│   ├── 化工原料
│   └── 农产品
├── 半成品 (Semi-finished Product)
├── 成品 (Finished Product)
├── 辅料 (Auxiliary Material)
├── 设备 (Equipment)
│   ├── 生产设备
│   └── 办公设备
└── 服务 (Service)
    ├── 物流服务
    ├── 咨询服务
    └── 维修服务
```

## 二、规律 (Laws)

### 2.1 供应链牛鞭效应

**定义**: 需求信息在供应链各环节传递时，逐级放大导致上游波动远大于下游的现象。

**公式**:
```
波动放大系数 = 上游需求方差 / 下游需求方差
```

**应对原则**:
- 信息共享：打通供应链各环节数据
- 缩短lead time：缩短采购和交付周期
- 稳定需求：建立长期合作关系

### 2.2 库存周转规律

**定义**: 库存从入库到出库的循环效率。

**核心指标**:
```
库存周转率 = 期间出库成本 / 平均库存
周转天数 = 365 / 周转率
```

**行业基准**:
- 快消品: 8-12次/年
- 制造业: 4-6次/年
- 原材料: 2-4次/年

### 2.3 供应商准时交付率 (OTD)

**定义**: 供应商在承诺日期内完成交付的比例。

**计算公式**:
```
OTD = 准时交付订单数 / 总订单数 × 100%
```

**等级划分**:
- 优秀: OTD ≥ 98%
- 良好: 95% ≤ OTD < 98%
- 合格: 90% ≤ OTD < 95%
- 不合格: OTD < 90%

### 2.4 采购价格弹性

**定义**: 采购量变化对采购价格的影响程度。

**规律**:
- 采购量与单价呈负相关
- 长期合作价格低于短期采购
- 集中采购可获得更高折扣

## 三、业务规则 (Rules)

### 3.1 供应商准入规则

| 条件 | 要求 |
|------|------|
| 资质要求 | 具备营业执照、相关行业许可证 |
| 质量体系 | 通过ISO 9001或同等质量体系认证 |
| 财务健康 | 资产负债率 < 70%，近两年盈利 |
| 产能要求 | 可满足订单峰值150%的产能 |
| 环保合规 | 符合当地环保法规要求 |

### 3.2 采购审批权限

| 采购金额 | 审批人 |
|----------|--------|
| < ¥10,000 | 采购员 |
| ¥10,000 - ¥50,000 | 采购经理 |
| ¥50,000 - ¥200,000 | 采购总监 |
| ¥200,000 - ¥1,000,000 | 副总经理 |
| > ¥1,000,000 | 总经理/董事会 |

### 3.3 采购流程规则

```
需求提出 → 供应商选择 → 询价比价 → 合同签订 → 订单下达 → 跟踪交付 → 验收入库 → 发票结算
```

**关键节点**:
1. **询价比价**: 询价供应商数量 ≥ 3家
2. **合同签订**: 金额 > ¥50,000 必须签订正式合同
3. **验收入库**: 质量检验合格后方可入库
4. **发票结算**: 发票、合同、验收单三单匹配

### 3.4 供应商绩效考核规则

**考核维度及权重**:

| 维度 | 权重 | 指标 |
|------|------|------|
| 质量 | 30% | 来料不良率、返工率、客诉次数 |
| 交付 | 30% | OTD准时率、订单变更响应速度 |
| 价格 | 20% | 与市场价格对比、年降幅度 |
| 服务 | 10% | 响应速度、技术支持能力 |
| 环保 | 10% | 环保合规、可持续发展 |

**等级评定**:
- A级 (≥90分): 战略供应商，优先合作
- B级 (75-89分): 合格供应商，正常合作
- C级 (60-74分): 观察供应商，辅导改进
- D级 (<60分): 不合格供应商，淘汰或替换

### 3.5 安全库存规则

**计算公式**:
```
安全库存 = (最大日用量 × 最长采购周期) - (平均日用量 × 平均采购周期)
```

**最低库存警示**:
- 触发补货: 库存 ≤ 安全库存
- 紧急补货: 库存 ≤ 安全库存 × 50%
- 断货风险: 库存 = 0

### 3.6 合同履约规则

- 付款账期: 默认月结30天，可协商
- 违约责任: 延迟交付按合同金额1%/天收取违约金
- 价格保护: 合同期内市场降价需同步调整
- 续约提前: 合同到期前60天启动续约评估

## 五、应用场景

1. **智能问答**: 基于本体知识回答采购流程、供应商管理等常见问题
2. **流程自动化**: 根据业务规则自动触发审批、提醒、预警
3. **数据分析**: 基于规律进行库存优化、供应商评估、价格预测
4. **知识推理**: 通过本体关系发现潜在供应商、识别供应链风险

## 六、核心业务场景 (MVP)

### 6.1 供应商风险管理

**场景**: 自动评估供应商风险等级，触发预警

**规则**:
```
IF 供应商绩效评分 < 60 THEN 标记为高风险
IF 供应商交付准时率 < 90% THEN 触发预警
IF 供应商财务异常 THEN 暂停合作并通知采购经理
```

### 6.2 智能询报价

**场景**: 自动向多个供应商发送询价请求

**规则**:
```
IF 采购金额 > 50000 THEN 询价供应商数量 >= 3
IF 物料类别 = 战略物资 THEN 只向A级供应商询价
```

### 6.3 库存优化建议

**场景**: 基于历史消耗数据给出补货建议

**规则**:
```
IF 当前库存 <= 安全库存 THEN 建议补货
IF 预测需求增长 > 20% THEN 建议提前备货
```
