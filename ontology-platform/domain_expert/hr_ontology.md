# 人力资源领域本体 (Human Resources Ontology)

## 1. HR核心概念

### 1.1 HR主体

| 概念 | 定义 | 关系 |
|------|------|------|
| 员工 (Employee) | 与组织建立劳动关系的个人 | 核心角色 |
| 管理者 (Manager) | 承担管理职责、领导团队的个人 | 管理角色 |
| HR专业人员 (HR Professional) | 从事人力资源管理工作的专业人员 | 专业角色 |
| 求职者 (Job Seeker) | 寻求就业机会的个人 | 潜在角色 |
| 候选人 (Candidate) | 通过筛选进入招聘流程的求职者 | 中间角色 |
| 外部顾问 (External Consultant) | 提供HR专业服务的外部人员 | 支持角色 |

### 1.2 HR职能

| 概念 | 定义 | 层次 |
|------|------|------|
| 人力资源规划 (HR Planning) | 预测组织人力需求并制定获取策略 | 战略层 |
| 招聘与配置 (Recruitment & Selection) | 吸引、选拔和配置人才 | 入口层 |
| 培训与开发 (Training & Development) | 提升员工能力、促进职业发展 | 发展层 |
| 绩效管理 (Performance Management) | 评估和改进员工工作表现 | 激励层 |
| 薪酬福利管理 (Compensation & Benefits) | 设计和管理员工薪酬体系 | 保障层 |
| 员工关系 (Employee Relations) | 维护和谐的劳动关系 | 维护层 |
| 职业健康管理 (Occupational Health) | 保障员工身心健康 | 保护层 |
| 人力资源信息系统 (HRIS) | HR数据管理与技术支持 | 支撑层 |

### 1.3 组织结构

```
董事会
    ↓
人力资源委员会
    ↓
HR总监 (CHRO)
    ↓
├── 人力资源规划部
├── 招聘与配置部
├── 培训与发展部
├── 绩效管理部
├── 薪酬福利部
├── 员工关系部
└── HR共享服务中心
```

### 1.4 劳动关系类型

```
标准劳动关系 ──→ 劳动合同制
               ──→ 劳务派遣
非标准劳动关系 ──→ 兼职
               ──→ 临时工
               ──→ 外包
特殊劳动关系 ──→ 实习生
               ──→ 退休返聘
               ──→ 外籍员工
```

## 2. 招聘规则

### 2.1 招聘原则

| 原则 | 说明 | 应用 |
|------|------|------|
| 公开公平 | 招聘信息公开、选拔标准公正 | 招聘渠道 |
| 竞争择优 | 通过竞争选择最优候选人 | 筛选机制 |
| 人岗匹配 | 岗位要求与候选人能力匹配 | 选拔标准 |
| 双向选择 | 尊重候选人和组织的选择权 | 决策过程 |
| 效率优先 | 在保证质量的前提下提高效率 | 流程优化 |

### 2.2 招聘流程

```
需求确认 → 招聘计划 → 渠道选择 → 信息发布
    ↓
简历筛选 → 初试 → 复试 → 终试
    ↓
体检 → 背景调查 → 录用通知 → 入职办理
```

### 2.3 招聘渠道

```
内部渠道              外部渠道
   │                    │
晋升 ──────┐          猎头 ──────┐
           │                    │
内部推荐 ──┼── 招聘渠道体系   网络招聘 ──┼
           │                    │
内部竞聘 ──┘          校园招聘 ──┘
                         │
                    招聘会 ──────┘
                    社交招聘
                    人才库
```

### 2.4 招聘规则引擎

```yaml
规则1: 招聘需求必须经过审批
  IF 用人部门提出招聘需求 THEN 必须经过HR审批
  IF 招聘人数 > 编制数 THEN 必须经过管理层审批

规则2: 候选人筛选必须多维度评估
  IF 筛选简历 THEN 必须评估：学历、工作经验、技能、薪资期望
  IF 面试 THEN 必须包含：专业能力、综合素质、价值观匹配

规则3: 背景调查必须覆盖关键岗位
  IF 岗位级别 ≥ 部门经理 THEN 必须进行背景调查
  IF 岗位涉及财务/安全 THEN 必须进行背景调查

规则4: 录用决策必须符合法规
  IF 候选人存在竞业限制 THEN 必须确认已解除
  IF 候选人存在劳动纠纷 THEN 必须进行风险评估
```

## 3. 培训规则

### 3.1 培训原则

| 原则 | 说明 | 应用 |
|------|------|------|
| 战略导向 | 培训服务于组织战略目标 | 培训规划 |
| 需求导向 | 基于岗位胜任力和绩效差距 | 培训设计 |
| 成人学习 | 尊重成人学习规律 | 方法选择 |
| 效果导向 | 注重培训转化和应用 | 评估体系 |
| 持续学习 | 建立终身学习机制 | 学习文化 |

### 3.2 培训体系

```
培训体系
    │
    ├── 新员工入职培训
    │       ├── 公司文化
    │       ├── 规章制度
    │       ├── 岗位职责
    │       └── 安全培训
    │
    ├── 在岗技能培训
    │       ├── 专业技能
    │       ├── 管理技能
    │       └── 通用技能
    │
    ├── 领导力发展
    │       ├── 高层管理培训
    │       ├── 中层管理培训
    │       └── 基层管理培训
    │
    └── 职业发展培训
            ├── 晋升培训
            ├── 轮岗培训
            └── 继任计划培训
```

### 3.3 培训方法

```
课堂培训 (Classroom Training)
    │
    ├── 讲授法 (Lecture)
    ├── 案例法 (Case Study)
    ├── 角色扮演 (Role Play)
    ├── 小组讨论 (Group Discussion)
    └── 工作坊 (Workshop)

实践培训 (Practical Training)
    │
    ├── 师徒制 (Mentoring)
    ├── 在岗培训 (OJT)
    ├── 轮岗实习 (Job Rotation)
    └── 项目锻炼 (Project Assignment)

数字化学习 (E-Learning)
    │
    ├── 在线课程 (Online Course)
    ├── 移动学习 (Mobile Learning)
    ├── 微课 (Micro-learning)
    └── 虚拟现实 (VR/AR Training)
```

### 3.4 培训规则引擎

```yaml
规则1: 培训需求必须基于分析
  IF 绩效评估显示能力差距 THEN 制定培训计划
  IF 新业务/新技术引入 THEN 评估培训需求

规则2: 培训计划必须分层分类
  IF 培训对象 = 高管 THEN 侧重战略领导力
  IF 培训对象 = 中层 THEN 侧重管理技能
  IF 培训对象 = 基层 THEN 侧重专业技能

规则3: 培训效果必须评估
  IF 培训时间 > 8小时 THEN 必须进行效果评估
  IF 培训费用 > 万元 THEN 必须进行效果评估
  
规则4: 培训记录必须归档
  IF 员工完成培训 THEN 更新培训档案
  IF 培训考核不合格 THEN 安排补训
```

## 4. 绩效管理规则

### 4.1 绩效管理原则

| 原则 | 说明 | 应用 |
|------|------|------|
| 战略对齐 | 绩效目标与组织战略一致 | 目标设定 |
| 客观公正 | 评价标准明确、评价过程透明 | 评估实施 |
| 持续沟通 | 绩效过程持续反馈和辅导 | 沟通机制 |
| 激励发展 | 绩效结果用于激励和发展 | 结果应用 |
| 改进提升 | 绩效管理促进持续改进 | 循环优化 |

### 4.2 绩效管理流程

```
绩效计划 → 绩效辅导 → 绩效评估 → 绩效反馈 → 结果应用
   ↑                                            ↓
   └──────────────── 持续改进 ←─────────────────┘
```

### 4.3 绩效考核方法

```
结果导向型
    │
    ├── 目标管理 (MBO)
    ├── 关键绩效指标 (KPI)
    └── 平衡计分卡 (BSC)

行为导向型
    │
    ├── 行为锚定等级评价 (BARS)
    ├── 行为观察评价 (BOS)
    └── 360度反馈

发展导向型
    │
    ├── 绩效棱镜
    └── 目标与关键成果 (OKR)
```

### 4.4 绩效管理规则引擎

```yaml
规则1: 绩效目标必须SMART
  IF 设定绩效目标 THEN 必须符合：具体、可衡量、可达成、相关、有时限
  IF 目标设定不符合SMART THEN 驳回重设

规则2: 绩效评估必须多维度
  IF 年度绩效评估 THEN 必须包含：业绩、能力、态度
  IF 管理人员 THEN 必须包含团队管理维度

规则3: 绩效结果必须强制分布
  IF 员工规模 > 100人 THEN 启用强制分布
  IF 强制分布 THEN 比例：A(10-20%)、B(60-70%)、C(10-20%)、D(0-5%)

规则4: 绩效结果必须反馈确认
  IF 完成绩效评估 THEN 必须与员工面谈确认
  IF 员工对结果有异议 THEN 开启申诉流程
  
规则5: 绩效结果必须与应用挂钩
  IF 绩效等级 = A THEN 年度调薪优先考虑
  IF 绩效等级 = D THEN 启动改进计划或淘汰流程
```

## 5. 薪酬管理规则

### 5.1 薪酬管理原则

| 原则 | 说明 | 应用 |
|------|------|------|
| 外部竞争性 | 薪酬水平具有市场竞争力 | 薪酬定位 |
| 内部公平性 | 薪酬与岗位价值和个人贡献匹配 | 薪酬结构 |
| 激励性 | 薪酬能够有效激励员工行为 | 薪酬设计 |
| 经济性 | 薪酬成本在可控范围内 | 成本控制 |
| 合法性 | 薪酬符合法律法规要求 | 合规管理 |

### 5.2 薪酬体系

```
薪酬体系
    │
    ├── 基本工资 (Base Salary)
    │       ├── 岗位工资
    │       └── 技能工资
    │
    ├── 绩效工资 (Performance Pay)
    │       ├── 月度绩效
    │       ├── 季度绩效
    │       └── 年度绩效
    │
    ├── 奖金 (Bonus)
    │       ├── 销售奖金
    │       ├── 项目奖金
    │       └── 年终奖金
    │
    ├── 福利 (Benefits)
    │       ├── 社会保险
    │       ├── 住房公积金
    │       ├── 商业保险
    │       └── 弹性福利
    │
    └── 长期激励 (Long-term Incentive)
            ├── 股票期权
            ├── 限制性股票
            └── 虚拟股权
```

### 5.3 薪酬结构

```
薪酬结构矩阵

职级        基本工资占比  绩效工资占比  奖金占比
─────────────────────────────────────
P1-P3       70%          15%          15%
P4-P6       60%          20%          20%
M1-M3       50%          25%          25%
M4+         40%          30%          30%
```

### 5.4 薪酬管理规则引擎

```yaml
规则1: 薪酬调整必须有依据
  IF 年度调薪 THEN 基于：绩效、市场、预算
  IF 晋升调薪 THEN 基于：新岗位薪酬范围
  
规则2: 薪酬必须保密
  IF 员工询问他人薪酬 THEN 告知保密政策
  IF 泄露薪酬信息 THEN 触发纪律处分

规则3: 加班工资计算必须合规
  IF 工作日加班 THEN 按基本工资150%计算
  IF 休息日加班 THEN 安排调休或200%计算
  IF 法定节假日加班 THEN 按300%计算

规则4: 薪酬发放必须及时准确
  IF 发薪日遇节假日 THEN 提前发放
  IF 薪酬计算错误 THEN 次月补发
  
规则5: 高管薪酬必须披露
  IF 上市公司 THEN 依法披露高管薪酬
  IF 非上市公司 THEN 报董事会审批
```

## 6. 员工关系规则

### 6.1 员工关系原则

| 原则 | 说明 | 应用 |
|------|------|------|
| 依法合规 | 遵守劳动法律法规 | 制度设计 |
| 平等尊重 | 尊重员工人格和权利 | 日常管理 |
| 沟通协商 | 建立有效沟通机制 | 关系维护 |
| 利益平衡 | 平衡组织和员工利益 | 决策考量 |
| 预防为主 | 预防和化解劳动争议 | 风险管理 |

### 6.2 员工关系管理

```
员工关系管理
    │
    ├── 劳动合同管理
    │       ├── 合同签订
    │       ├── 合同变更
    │       ├── 合同续签
    │       └── 合同解除
    │
    ├── 劳动争议处理
    │       ├── 协商调解
    │       ├── 仲裁申请
    │       └── 诉讼应对
    │
    ├── 员工沟通
    │       ├── 正式沟通
    │       ├── 非正式沟通
    │       └── 员工满意度调查
    │
    ├── 企业文化
    │       ├── 价值观建设
    │       ├── 文化活动
    │       └── 员工关怀
    │
    └── 纪律管理
            ├── 规章制度
            ├── 违纪处理
            └── 申诉处理
```

### 6.3 员工关系规则引擎

```yaml
规则1: 劳动合同必须依法签订
  IF 建立劳动关系 THEN 一个月内签订书面劳动合同
  IF 合同到期 THEN 提前一个月启动续签流程
  
规则2: 试用期管理必须合规
  IF 劳动合同期限 ≥ 3年 THEN 试用期 ≤ 6个月
  IF 劳动合同期限 < 3年 THEN 试用期按比例缩短
  IF 试用期不符合录用条件 THEN 解除合同

规则3: 解除劳动合同必须有法定理由
  IF 协商解除 THEN 支付经济补偿
  IF 员工过错解除 THEN 无补偿但需有证据
  IF 客观情况发生重大变化解除 THEN 需先协商

规则4: 经济性裁员必须程序合规
  IF 裁员 ≥ 20人 THEN 提前30天报告工会
  IF 裁员方案 THEN 需向劳动部门报告
  
规则5: 竞业限制必须支付补偿
  IF 约定竞业限制 THEN 离职后按月支付补偿
  IF 补偿标准 < 离职前12个月平均工资30% THEN 约定无效
```

## 7. 合规管理规则

### 7.1 HR合规领域

| 领域 | 主要法规 | 违规风险 |
|------|----------|----------|
| 劳动合同 | 《劳动合同法》 | 双倍工资、违法解除赔偿 |
| 工时与休假 | 《劳动法》《职工带薪年休假条例》 | 加班费、未休年假补偿 |
| 薪酬社保 | 《社会保险法》《住房公积金管理条例》 | 补缴、滞纳金、罚款 |
| 女职工保护 | 《女职工劳动保护特别规定》 | 行政处罚、赔偿 |
| 未成年工保护 | 《未成年工特殊保护规定》 | 行政处罚 |
| 职业安全 | 《职业病防治法》《安全生产法》 | 行政处罚、刑事责任 |

### 7.2 合规检查清单

```
入职管理
    □ 身份信息核验
    □ 学历证书验证
    □ 背景调查（如适用）
    □ 体检（如适用）
    □ 劳动合同签订
    □ 规章制度告知
    □ 社保公积金开户

在职管理
    □ 合同续签提醒
    □ 社保公积金及时缴纳
    □ 加班审批与记录
    □ 年休假安排与记录
    □ 工资按时足额发放
    □ 劳动防护用品配备

离职管理
    □ 离职面谈
    □ 工作交接
    □ 离职证明开具
    □ 社保公积金转移
    □ 竞业限制启动（如有）
    □ 档案转移
```

### 7.3 合规规则引擎

```yaml
规则1: 工时制度必须合规申报
  IF 实行标准工时 THEN 每日8小时、每周40小时
  IF 实行综合工时 THEN 需劳动部门审批
  IF 实行不定时工时 THEN 需劳动部门审批

规则2: 社保缴纳必须合规
  IF 员工入职 THEN 30日内缴纳社保
  IF 社保基数不按实际工资 THEN 补缴+滞纳金
  
规则3: 劳动保护必须到位
  IF 高温作业 THEN 发放高温津贴
  IF 接触职业病危害因素 THEN 定期体检
  IF 女职工孕期/产期/哺乳期 THEN 调整岗位或减轻工作量

规则4: 档案管理必须规范
  IF 员工档案丢失 THEN 承担相应责任
  IF 档案转移延迟 THEN 影响员工权益
```

## 8. 组织与人才发展规则

### 8.1 组织发展概念

| 概念 | 定义 | 关系 |
|------|------|------|
| 组织架构 (Organizational Structure) | 组织的权责体系和工作分工 | 基础结构 |
| 岗位体系 (Job System) | 岗位分类、职责和任职资格 | 人员基础 |
| 职级体系 (Career Path) | 职业发展通道和等级 | 发展路径 |
| 编制管理 (Headcount Management) | 人员数量和结构控制 | 资源约束 |
| 人才梯队 (Talent Pipeline) | 关键岗位的后备人才 | 继任保障 |

### 8.2 人才发展流程

```
人才发展战略
    │
    ├── 人才盘点
    │       ├── 数量盘点
    │       ├── 结构盘点
    │       └── 能力盘点
    │
    ├── 人才识别
    │       ├── 绩效数据分析
    │       ├── 能力评估
    │       ├── 潜力评估
    │       └── 360度反馈
    │
    ├── 人才发展规划
    │       ├── 职业发展规划
    │       ├── 继任计划
    │       └── 高潜培养计划
    │
    └── 人才培养
            ├── 导师辅导
            ├── 挑战性任务
            ├── 跨部门轮岗
```

### 8.3 组织发展规则引擎


## 9. HR指标体系

### 9.1 招聘效率指标

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|

| 指标 | 计算公式 | 目标值 |

### 9.4 人员流动指标

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|

```yaml
规则4: 远程办公必须明确管理规范
  IF 允许远程办公 THEN 明确工作时间、响应要求
  IF 涉及数据安全 THEN 签署保密协议
```

```yaml
  IF 模型准确率低于阈值 THEN 重新建模

规则3: 数据驱动决策必须有依据
  IF 基于数据分析做决策 THEN 确保数据质量
  IF 数据样本不足 THEN 谨慎下结论

规则4: 分析结果必须可解释
  IF 给出分析结论 THEN 提供业务解释
  IF 相关性不等于因果性 THEN 明确说明
```
