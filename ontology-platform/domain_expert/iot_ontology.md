# 物联网领域本体设计

## 概述

本文档定义物联网（Internet of Things, IoT）领域的核心本体模型，用于指导智能家居、工业互联网、智慧城市等场景的知识库构建和智能化应用。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 物联网设备 | IoT Device | 具备感知能力和网络通信能力的智能硬件 | 设备ID、名称、类型、厂商、固件版本、状态 |
| 传感器 | Sensor | 用于采集物理环境数据的装置 | 传感器ID、测量类型、精度、量程、采样率 |
| 执行器 | Actuator | 执行控制指令的装置 | 执行器ID、动作类型、响应时间、控制范围 |
| 网关 | Gateway | 连接终端设备与云平台的中间设备 | 网关ID、协议类型、带宽、边缘计算能力 |
| 物联网平台 | IoT Platform | 负责设备管理、数据采集、规则引擎的云平台 | 平台ID、名称、厂商、支持协议 |
| 数据流 | Data Stream | 设备产生的连续数据序列 | 流ID、数据类型、时间戳、采样频率 |
| 数字孪生 | Digital Twin | 物理设备的虚拟数字映射 | 孪生ID、对应设备、模型精度、同步频率 |
| 边缘节点 | Edge Node | 具备本地计算能力的边缘设备 | 节点ID、算力、存储、延迟要求 |

### 1.2 关系定义

```
传感器 ⊚ 部署于 [物联网设备] (多对一)
执行器 ⊚ 集成于 [物联网设备] (多对一)
物联网设备 ⊚ 接入 [网关] (多对一)
网关 ⊚ 连接 [物联网平台] (多对一)
物联网设备 ⊚ 产生 [数据流] (一对多)
数据流 ⊚ 汇聚至 [物联网平台] (多对一)
物联网设备 ⊚ 映射 [数字孪生] (一对一)
边缘节点 ⊚ 部署于 [网关] (多对一)
物联网平台 ⊚ 托管 [数字孪生] (一对多)
```

## 二、规律 (Laws)

### 2.1 摩尔定律在物联网硬件中的应用

**定义**: 物联网芯片的计算能力每18-24个月翻倍，同时成本下降。

**公式**:
```
计算能力 = 基础算力 × 2^(年份差/24)
单位成本 = 基础成本 × 2^(-年份差/24)
```

**行业影响**:
- 设备智能化成本持续下降
- 边缘计算能力不断增强
- 嵌入式AI成为可能

### 2.2 设备生命周期规律

**定义**: 物联网设备从部署到淘汰的典型周期。

**阶段划分**:
```
部署期(0-6月) → 成长期(6-18月) → 成熟期(18-36月) → 衰退期(36-60月) → 淘汰期(>60月)
```

**规律**:
- 部署期故障率较高（ infant mortality）
- 成熟期故障率稳定在最低水平
- 衰退期故障率随元器件老化上升

### 2.3 数据产生速率规律

**定义**: 不同类型物联网设备的数据产生速率。

**典型速率**:

| 设备类型 | 数据类型 | 典型速率 |
|----------|----------|----------|
| 温湿度传感器 | 数值型 | 1-60 KB/天 |
| 视频监控摄像头 | 视频流 | 1-20 GB/天 |
| 智能电表 | 数值型 | 10-100 KB/天 |
| 工业振动传感器 | 时序数据 | 1-100 MB/天 |
| GPS追踪器 | 位置数据 | 100-500 KB/天 |

**公式**:
```
日均数据量 = 采样频率 × 数据包大小 × 3600 × 24 / 压缩比
```

### 2.4 网络覆盖与功耗关系

**定义**: 通信距离与设备功耗的数学关系。

**公式**:
```
功耗 ∝ 距离^n (n=2~4，取决于环境)
```

**实际应用**:
- 短距离协议（BLE）: 功耗最低，覆盖<10m
- 中距离协议（Zigbee）: 功耗中等，覆盖10-100m
- 长距离协议（NB-IoT）: 功耗较高，覆盖可达10km+

### 2.5 边缘-云协同定律

**定义**: 数据处理位置选择的成本-延迟权衡。

**公式**:
```
总成本 = 传输成本 + 计算成本 + 延迟成本
最佳处理位置 = f(数据量、实时性要求、带宽成本、边缘算力)
```

**决策原则**:
- 毫秒级响应: 边缘处理
- 秒级响应: 边缘+云协同
- 分钟级响应: 云端处理

## 三、业务规则 (Rules)

### 3.1 设备接入规则

| 条件 | 要求 |
|------|------|
| 身份认证 | 设备必须具备唯一标识（MAC/序列号）+ 数字证书 |
| 协议兼容 | 支持MQTT或CoAP等标准IoT协议 |
| 安全能力 | 支持TLS/SSL加密传输，具备固件签名验证 |
| 资源要求 | 具备至少128KB RAM可用于安全协议 |
| 厂商认证 | 列入平台认证设备清单或通过互操作性测试 |

### 3.2 设备命名规范

```
[用途前缀]_[设备类型]_[序列号]_[位置]
例如：
- sensor_temp_001_livingroom（客厅温度传感器）
- actuator_valve_001_kitchen（厨房水阀执行器）
- camera_security_001_entrance（门口安防摄像头）
```

### 3.3 数据采集规则

**采样频率配置**:

| 数据类型 | 最小采样频率 | 推荐采样频率 | 最大采样频率 |
|----------|--------------|--------------|--------------|
| 温度 | 1次/分钟 | 1次/分钟 | 1次/秒 |
| 湿度 | 1次/分钟 | 1次/分钟 | 1次/秒 |
| 压力 | 1次/秒 | 10次/秒 | 1KHz |
| 视频 | 1fps | 25fps | 60fps |
| 振动 | 100Hz | 1KHz | 10KHz |
| 位置 | 1次/分钟 | 1次/秒 | 10Hz |

**数据上传策略**:
- 实时数据: 立即上传
- 批量数据: 本地缓存，5分钟上传一次
- 异常数据: 立即上传并触发告警

### 3.4 设备状态管理规则

**设备状态定义**:

| 状态 | 描述 | 判定条件 |
|------|------|----------|
| 在线 | 设备正常运行，可通信 | 最近一次心跳 < 5分钟 |
| 离线 | 设备无法通信 | 最近一次心跳 > 5分钟 |
| 告警 | 设备检测到异常 | 传感器数值超出阈值 |
| 升级中 | 设备正在更新固件 | 固件升级进行中 |
| 休眠 | 设备进入低功耗模式 | 配置为低功耗模式且处于休眠周期 |

**状态转换规则**:
```
在线 → 离线: 连续5分钟无心跳
离线 → 在线: 收到设备主动上报
在线 → 告警: 任一指标超出阈值
告警 → 在线: 指标恢复正常并保持1分钟
任意状态 → 升级中: 收到OTA升级指令
升级中 → 在线: 升级成功并重新上线
```

### 3.5 安全规则

**设备安全要求**:

| 类别 | 要求 | 优先级 |
|------|------|--------|
| 身份认证 | 设备证书双向认证 | 必须 |
| 传输加密 | TLS 1.2及以上 | 必须 |
| 固件安全 | 固件签名验证、安全启动 | 必须 |
| 访问控制 | 基于角色的访问控制(RBAC) | 必须 |
| 日志审计 | 操作日志留存≥180天 | 必须 |
| 漏洞管理 | 已知漏洞90天内修复 | 高 |
| 数据安全 | 敏感数据加密存储 | 高 |

**异常检测规则**:
- 设备尝试连接陌生网络 → 告警
- 短时间内大量异常登录尝试 → 锁定账户
- 设备流量异常（过高或过低）→ 告警
- 固件版本低于安全基线 → 提醒升级

### 3.6 数字孪生管理规则

**孪生同步策略**:

| 同步类型 | 触发条件 | 同步延迟 | 适用场景 |
|----------|----------|----------|----------|
| 实时同步 | 设备状态变化 | <1秒 | 告警、监控 |
| 批量同步 | 定时任务 | <1分钟 | 历史数据 |
| 事件同步 | 特定事件触发 | <5秒 | 故障分析 |

**孪生精度等级**:

| 等级 | 描述 | 数据精度 | 模型复杂度 |
|------|------|----------|------------|
| L1 | 基础映射 | 5分钟级 | 静态模型 |
| L2 | 状态同步 | 1秒级 | 动态模型 |
| L3 | 实时联动 | 毫秒级 | 仿真模型 |
| L4 | 预测推演 | 实时 | AI预测模型 |

### 3.7 告警规则

**告警级别定义**:

| 级别 | 名称 | 描述 | 响应时间 | 通知方式 |
|------|------|------|----------|----------|
| P0 | 紧急 | 设备离线、安全告警 | <5分钟 | 电话+短信+邮件 |
| P1 | 严重 | 核心指标异常 | <15分钟 | 短信+邮件 |
| P2 | 一般 | 次要指标异常 | <1小时 | 邮件+App推送 |
| P3 | 提示 | 预警提醒 | <24小时 | App推送 |

**告警收敛规则**:
- 同一设备5分钟内相同告警 → 合并为1条
- 同一设备5分钟内不同告警 → 合并为告警组
- 告警恢复确认后 → 自动关闭原始告警

## 五、应用场景

### 5.1 智能家居

- **设备互联**: 通过网关实现全屋设备联动
- **场景自动化**: 根据时间、位置、环境自动触发场景
- **能耗管理**: 智能调节家电功耗，优化电费支出
- **家庭安全**: 异常入侵、火警水浸即时告警

### 5.2 工业互联网 (IIoT)

- **预测性维护**: 通过振动、温度数据分析设备故障趋势
- **质量追溯**: 全流程数据采集，问题追溯到根因
- **产能优化**: 实时监控产线效率，优化生产排程
- **安全监控**: 危险区域入侵检测、有害气体监测

### 5.3 智慧城市

- **交通治理**: 实时交通流量分析，智能信号灯调度
- **环境监测**: 空气质量、噪音、水质全面感知
- **公共设施**: 路灯、垃圾桶、井盖智能管理
- **应急响应**: 突发事件快速感知与资源调度

### 5.4 智慧农业

- **精准种植**: 土壤湿度、光照、气象数据指导灌溉施肥
- **畜牧监测**: 动物体温、活动量预警疾病
- **冷链物流**: 全程温湿度监控，保障农产品品质
- **产量预测**: 历史数据与AI模型预测收获时间
