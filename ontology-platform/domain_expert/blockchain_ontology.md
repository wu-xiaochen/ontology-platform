# 区块链领域本体设计

## 概述

本文档定义区块链领域的核心本体模型，用于指导区块链知识库构建和智能化应用。涵盖区块链技术原理、加密资产、DeFi、NFT、Web3等细分领域。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

#### 1.1.1 区块链基础架构

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 区块链 | Blockchain | 分布式账本技术，由按时间顺序连接的区块组成的去中心化数据库 | 区块高度、创世区块、共识算法、分叉状态 |
| 区块 | Block | 区块链上的数据单元，包含交易信息和元数据 | 区块哈希、上一区块哈希、时间戳、交易数量、区块大小 |
| 交易 | Transaction | 区块链上价值转移的记录 | 交易哈希、发送方、接收方、金额、手续费、状态 |
| 账户 | Account | 区块链上的地址实体 | 地址、公钥、私钥、余额、Nonce |
| 智能合约 | Smart Contract | 部署在区块链上的自动执行代码 | 合约地址、合约字节码、ABI、Gas限制 |

#### 1.1.2 共识机制

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 共识机制 | Consensus Mechanism | 分布式节点就账本状态达成一致的算法 | 机制类型、吞吐量、TPS、出块时间、能耗 |
| 工作量证明 | Proof of Work (PoW) | 通过计算难题竞争出块权的共识机制 | 算力难度、矿工收益、区块奖励 |
| 权益证明 | Proof of Stake (PoS) | 根据持币量选择出块者的共识机制 | 质押数量、质押期限、验证者数量 |
| 委托权益证明 | Delegated PoS (DPoS) | 持币者投票选出的代表轮流出块 | 受托人数量、投票机制、惩罚机制 |
| 实用拜占庭容错 | PBFT | 联盟链常用的共识算法 | 节点数量、容错能力、交易确认时间 |

#### 1.1.3 加密资产

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 加密货币 | Cryptocurrency | 基于密码学原理发行的数字资产 | 名称、符号、总供应量、市值、价格 |
| 比特币 | Bitcoin (BTC) | 第一个去中心化加密货币 | 区块奖励、减半周期、2100万上限 |
| 以太坊 | Ethereum (ETH) | 支持智能合约的区块链平台 | Gas机制、EVM、升级路线 |
| 稳定币 | Stablecoin | 锚定法币或资产的加密货币 | 锚定机制、抵押类型、发行方 |
| 平台币 | Platform Token | 区块链平台发行的功能性代币 | 用途、销毁机制、治理权 |
| 迷因币 | Meme Coin | 基于互联网文化的加密货币 | 社区驱动、投机性强、无实际用例 |

#### 1.1.4 金融概念

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 去中心化交易所 | DEX | 基于智能合约的点对点交易平台 | 交易对、流动性、做市商机制 |
| 流动性池 | Liquidity Pool | DEX中提供流动性的资金池 | 流动性深度、APY、滑点 |
| 借贷协议 | Lending Protocol | 去中心化借贷平台 | 存款利率、借款利率、抵押率、清算阈值 |
| 收益聚合器 | Yield Aggregator | 自动优化收益的DeFi协议 | 收益策略、聚合收益、风险等级 |
| 合成资产 | Synthetic Asset | 模拟其他资产价值的代币 | 抵押品类型、追踪资产、预言机 |

#### 1.1.5 NFT与数字资产

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 非同质化代币 | NFT | 代表独特资产的代币标准 | 元数据、所有权、转让记录 |

| 域名 | Domain Name | 区块链域名服务 | 解析记录、续费周期、交易市场 |

#### 1.1.6 钱包与安全

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 加密钱包 | Crypto Wallet | 管理加密资产私钥的工具 | 钱包类型、支持币种、备份方式 |
| 热钱包 | Hot Wallet | 连接互联网的钱包 | 便捷性、风险等级 |
| 冷钱包 | Cold Wallet | 离线存储的钱包 | 安全性、存储介质 |

#### 1.1.7 挖矿与节点

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 矿工 | Miner | 参与PoW共识的节点 | 算力、矿池、收益 |
| 矿池 | Mining Pool | 矿工协作挖矿的组织 | 分配方式、手续费、算力占比 |
| 验证者 | Validator | PoS/DPoS共识中的出块节点 | 质押量、在线率、惩罚记录 |
| 全节点 | Full Node | 完整同步所有区块数据的节点 | 存储需求、带宽要求 |
| 轻节点 | Light Node | 只同步区块头的节点 | 存储需求、验证方式 |
| RPC节点 | RPC Node | 提供API服务的区块链节点 | 端点URL、速率限制 |

## 二、规律 (Laws)

### 2.1 比特币减半周期

**定义**: 比特币区块奖励每21万个区块（约4年）减半一次的机制。

**规律**:
- 2009年: 50 BTC/区块
- 2012年: 25 BTC/区块
- 2016年: 12.5 BTC/区块
- 2020年: 6.25 BTC/区块
- 2024年: 3.125 BTC/区块

**市场影响**:
- 减半前通常有预期上涨
- 减半后供应减少，长期看涨
- 历史上减半后12-18个月出现牛市

### 2.2 Gas费波动规律

**定义**: 以太坊网络拥堵时Gas费用自发上涨的机制。

**影响因素**:
```
Gas费 = Base Fee + Priority Fee

Base Fee = 前一区块利用率 × 调整系数
```

**规律**:
- 区块利用率 > 50% → Base Fee 上升
- 区块利用率 < 50% → Base Fee 下降
- 网络活跃时Gas费可达数十至数百Gwei
- L2网络可显著降低Gas成本

### 2.3 流动性挖矿收益衰减

**定义**: 流动性挖矿激励随着时间推移和资金流入而逐步降低。

**规律**:
- 项目早期APY可达数百甚至数千%
- 随着TVL增长，APY呈指数下降
- 需不断寻找新项目或策略轮换
- 高APY通常伴随高IL（无常损失）风险

### 2.4 清算机制

**定义**: 抵押不足时触发资产强制平仓的风险控制机制。

**清算阈值公式**:
```
清算触发 = 抵押物价值 / 借款价值 < 最低抵押率

清算 penalty = 借款金额 × 清算费率 (通常5-10%)
```

**规律**:
- 市场剧烈波动时易触发连环清算
- 清算发生后抵押物以折扣价出售
- 闪电贷可利用价格差进行套利清算

### 2.5 三角权衡 (Blockchain Trilemma)

**定义**: 区块链无法同时实现去中心化、安全性、可扩展性三个目标。

| 优先目标 | 牺牲目标 | 代表公链 |
|----------|----------|----------|
| 去中心化 + 安全 | 可扩展性 | Bitcoin, Ethereum |
| 安全 + 可扩展性 | 去中心化 | Solana, Avalanche |
| 去中心化 + 可扩展性 | 安全性 | 多数测试网 |

## 三、业务规则 (Rules)

### 3.1 交易规则

#### 3.1.1 交易生命周期

| 阶段 | 描述 | 状态 |
|------|------|------|
| 创建 | 用户签名交易并广播 | pending |
| 内存池 | 交易等待确认 (Mempool) | pending |
| 打包 | 矿工/验证者将交易纳入区块 | confirmed |
| 最终确认 | 获得足够区块确认数 | finality |

#### 3.1.2 交易确认规则

| 链类型 | 安全确认数 | 预计时间 |
|--------|------------|----------|
| Bitcoin | 6个区块 | ~60分钟 |
| Ethereum | 12-15个区块 (PoW) | ~3分钟 |
| Ethereum (PoS) | 2个Epoch (~12分钟) | ~12分钟 |
| L2 Rollup | 1个批次 + 7天挑战期 | ~7天 (最终) |
| 联盟链 | 1-3个区块 | 秒级 |

#### 3.1.3 交易费用规则

| 链/协议 | 计费方式 | 单位 |
|--------|----------|------|
| Bitcoin | sat/vB (聪/字节) | 字节大小 |
| Ethereum | Gas × Gwei | 执行步骤 |
| Solana | 每笔固定或Compute Units | CU消耗 |
| L2 | 与L1类似但更低 | Gas |

### 3.2 质押规则

#### 3.2.1 PoS质押要求

| 链 | 最低质押量 | 解锁期 | 惩罚机制 |
|----|------------|--------|----------|
| Ethereum | 32 ETH | 约18小时 | Slashing (扣押) |
| Solana | 1 SOL | ~2-3天 | Deactivation |
| Avalanche | 2000 AVAX | 约2周 | Cut |
| Cosmos | 1 ATOM | 21天 | Slashing |

#### 3.2.2 委托质押规则

| 规则 | 说明 |
|------|------|
| 委托门槛 | 通常低于直接质押 (如Solana 0.01 SOL) |
| 收益分配 | 验证者收取佣金 (通常5-20%) |
| 惩罚传导 | 验证者被罚没时，委托人也受影响 |
| 重新委托 | 可随时转移委托无需等待解锁 |

### 3.3 DeFi借贷规则

#### 3.3.1 借款参数

| 参数 | 说明 | 常见值 |
|------|------|--------|
| 抵押率 | 借款额/抵押价值上限 | 50-80% |
| 清算阈值 | 触发清算的抵押率 | 75-85% |
| 借款利率 | 借入资产的利率 | 浮动 |
| 存款利率 | 存入资产的收益率 | 浮动 |
| 清算惩罚 | 被清算时的额外费用 | 5-10% |

#### 3.3.2 主流协议参数

| 协议 | 抵押率 | 清算阈值 | 清算惩罚 |
|------|--------|----------|----------|
| Aave | 50-80% | 75% | 5% |
| Compound | 50-75% | 5-10% | 折价10% |
| MakerDAO | 130% (最低) | 150% | 13% |

### 3.4 NFT交易规则

#### 3.4.1 交易方式

| 方式 | 说明 | 优势 |
|------|------|------|
| 拍卖 (Auction) | 竞价成交 | 价格发现 |
| 荷兰拍 (Dutch Auction) | 逐级降价 | 快速出售 |
| 固定价格 | 一口价 | 简单确定 |
| 报价 (Offer) | 买方主动报价 | 灵活性 |

#### 3.4.2 版税规则

| 平台 | 默认版税 | 可自定义 |
|------|----------|----------|
| OpenSea | 2.5% | 可调整 |
| Blur | 0% | 强制 |
| Magic Eden | 2% | 可调整 |
| Foundation | 10% | 固定 |

### 3.5 钱包安全规则

#### 3.5.1 私钥管理原则

| 原则 | 说明 |
|------|------|
| 不触网 | 私钥/助记词绝不触网 |
| 分散存储 | 多地备份，避免单点失败 |
| 物理安全 | 防火防水防盗 |
| 不截屏 | 禁止截图保存助记词 |
| 不分享 | 官方不会索要私钥 |

#### 3.5.2 交易签名安全

| 风险类型 | 防护措施 |
|----------|----------|
| 授权攻击 | 检查授权金额和合约地址 |
| 假代币 | 核对合约地址 |
| 钓鱼网站 | 确认域名正确 |
| 恶意合约 | 使用安全插件 (如Revoke.cash) |

## 四、风险评估模型

### 4.1 智能合约风险

| 风险类型 | 描述 | 评估方法 |
|----------|------|----------|
| 重入攻击 | 合约调用递归导致资产被盗 | 代码审计、形式化验证 |
| 整数溢出 | 数值计算超出范围 | SafeMath、使用检查库 |
| 访问控制 | 未授权访问关键函数 | 权限修饰符检查 |
| 闪电贷攻击 | 利用大额借款操纵价格 | 价格预言机、TWAP |
| 合约升级风险 | 管理员可修改合约逻辑 | 多签、时间锁 |

### 4.2 DeFi风险评级

| 风险等级 | 特征 | 评估维度 |
|----------|------|----------|
| 低风险 | 成熟协议、审计完备、TVL高 | 代码审计、TVL、运行时间 |
| 中风险 | 新兴协议、有审计、TVL中等 | 团队背景、机制设计 |
| 高风险 | 无审计、匿名团队、新上线 | 代币经济学、激励机制 |

### 4.3 投资风险指标

| 指标 | 定义 | 评估标准 |
|------|------|----------|
| FDV | 完全稀释估值 | 市值/流通量比 |
| MC/FDV | 成熟度指标 | 越低越好 |
| 流动性深度 | 订单簿深度 | 滑点容忍度 |
| 波动率 | 价格变动幅度 | 历史波动 |
| 智能合约风险 | 协议安全性 | 审计次数、漏洞历史 |
