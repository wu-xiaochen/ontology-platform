# 金融风控领域本体设计

## 概述

本文档定义金融风控领域的核心本体模型，用于指导风险管理知识库构建和智能化应用。涵盖银行、消费金融、互联网金融等领域的信用风险、市场风险、操作风险管控。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

#### 1.1.1 客户主体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 借款人 | Borrower | 向金融机构申请贷款的法人或自然人 | 客户编号、姓名/名称、证件类型、证件号码、年龄、学历、职业 |
| 保证人 | Guarantor | 为借款人提供连带责任担保的第三方 | 客户编号、担保金额、担保期限、与借款人关系 |
| 抵押人 | Mortgagor | 提供抵押物作为债务担保的当事人 | 客户编号、抵押物类型、抵押物评估值、抵押率 |
| 配偶 | Spouse | 借款人的配偶（共同还款人） | 客户编号、姓名、证件类型、证件号码 |
| 实际控制人 | Actual Controller | 对企业实际行使控制权的人 | 客户编号、姓名、持股比例、任职情况 |

#### 1.1.2 产品定义

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 贷款产品 | Loan Product | 金融机构向客户提供的信贷产品 | 产品编号、产品名称、产品类型、额度范围、期限范围、利率范围、适用客群 |
| 信用卡产品 | Credit Card Product | 具有透支功能的银行卡产品 | 产品编号、产品名称、品牌、额度等级、年费政策、费率规则 |
| 理财产品 | Financial Product | 金融机构发行的收益凭证 | 产品编号、产品名称、风险等级、预期收益、期限、起投金额 |
| 担保产品 | Guarantee Product | 第三方提供的债务担保服务 | 产品编号、担保类型、担保费率、反担保要求 |

#### 1.1.3 交易与账户

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 贷款账户 | Loan Account | 借款人名下用于核算贷款本息的账户 | 账户编号、客户编号、产品编号、贷款金额、利率、剩余本金、账户状态 |
| 还款记录 | Repayment Record | 借款人按期归还本息的记录 | 记录编号、账户编号、还款日期、应还本金、应还利息、实际还款额、还款状态 |
| 逾期记录 | Overdue Record | 借款人未按期还款的记录 | 记录编号、账户编号、逾期天数、逾期金额、逾期阶段、催收记录 |
| 交易流水 | Transaction Flow | 客户账户的资金往来记录 | 流水编号、账户编号、交易时间、交易金额、交易类型、交易对手 |

#### 1.1.4 风控指标

| 概念 | 英文 | 定义 | 计算方式/说明 |
|------|------|------|---------------|
| 信用评分 | Credit Score | 反映客户信用风险的量化评分 | 基于客户画像、行为数据、还款记录等综合计算 |
| 风险等级 | Risk Grade | 综合评估后的风险分级 | 通常分为A/B/C/D/E五级或AAA/AA/A/BBB/BB/B六级 |
| 负债率 | Debt Ratio | 负债总额占收入的比例 | 负债总额 / 月收入 × 100% |
| 贷存比 | Loan-to-Deposit Ratio | 贷款余额与存款余额的比率 | 贷款余额 / 存款余额 × 100% |
| 拨备覆盖率 | Provision Coverage | 贷款损失准备与不良贷款的比例 | 贷款损失准备 / 不良贷款余额 × 100% |
| 不良率 | NPL Ratio | 不良贷款占全部贷款的比例 | 不良贷款余额 / 贷款总余额 × 100% |
| 损失率 | Loss Rate | 预期损失占贷款本金的比例 | 预期损失 / 贷款本金 × 100% |
| 在途金额 | Outstanding Amount | 当前未还本金加利息 | 剩余本金 + 未还利息 + 罚息 |
| 最大回撤 | Max Drawdown | 客户历史逾期最严重的程度 | 历史最严重逾期天数或金额 |

### 1.4 风险等级划分

| 等级 | 名称 | 评分区间 | 风险特征 | 管控策略 |
|------|------|----------|----------|----------|
| A | 优质 | 750-850 | 收入稳定、信用记录优秀、负债低 | 快速审批、优惠利率 |
| B | 良好 | 650-749 | 收入较稳定、有少量负债 | 正常审批、标准利率 |
| C | 一般 | 550-649 | 收入波动、信用记录一般 | 审慎审批、较高利率 |

| E | 高风险 | <450 | 严重逾期、多头借贷 | 拒批或抵押担保 |

## 二、规律 (Laws)

### 2.1 风险定价原则

**定义**: 风险越高，定价越高。金融机构通过利率覆盖预期损失和运营成本。

**公式**:
```
贷款利率 = 无风险利率 + 风险溢价 + 运营成本
风险溢价 = 预期损失率 × (1 + 目标利润率)
```

**应用**:
- 低风险客户可享受利率优惠
- 高风险客户需承担更高成本以覆盖风险
- 差异化定价实现风险与收益平衡

### 2.2 大数法则

**定义**: 在随机事件的大量重复出现中，往往呈现几乎必然的统计规律。

**风控应用**:
- 通过大量样本分散个体风险
- 足够的历史数据可准确预测违约概率
- 样本量越大，风险评估越精准

**业务启示**:
- 单一客户违约不影响整体资产质量
- 资产组合规模需达到风险分散的临界点
- 持续积累数据是风控能力提升的基础

### 2.3 信贷周期理论

**定义**: 信贷市场存在周期性波动，与经济周期紧密相关。

**周期阶段**:

| 阶段 | 特征 | 风控策略 |
|------|------|----------|
| 扩张期 | 信贷门槛降低、竞争加剧、不良率下降 | 适度收紧、优化准入 |
| 紧缩期 | 信贷收紧、风险暴露、不良率上升 | 加强审批、降低额度 |
| 调整期 | 市场出清、风险出清、资产质量改善 | 稳定策略、等待复苏 |

### 2.4 巴塞尔协议核心原则

**定义**: 国际银行监管的核心框架，确立资本充足率、风险管理等标准。

**三大支柱**:

1. **最低资本要求**: 核心一级资本充足率≥4.5%，一级资本充足率≥6%，总资本充足率≥8%
2. **监督检查**: 监管机构对银行风险进行评估和监督
3. **市场纪律**: 信息披露提高透明度

## 三、业务规则 (Rules)

### 3.1 贷前风控规则

#### 3.1.1 准入规则

| 准入维度 | 规则描述 | 阈值 |
|----------|----------|------|
| 年龄 | 借款人年龄要求 | 18-60岁（具体产品可能不同） |
| 信用记录 | 征信报告查询 | 近2年无严重逾期（连三累六） |
| 收入 | 月收入要求 | ≥月还款额的2倍 |
| 负债 | 负债率要求 | ≤50%（个人）/ ≤60%（企业） |
| 工作稳定性 | 在现单位工作时长 | ≥6个月/1年 |
| 经营年限 | 企业成立时间 | ≥2年 |

#### 3.1.2 反欺诈规则

| 规则类型 | 规则描述 | 处置方式 |
|----------|----------|----------|
| 身份欺诈 | 证件号码与姓名不匹配 | 拒绝 |
| 虚假信息 | 收入证明、流水造假 | 拒绝 |
| 多头借贷 | 同时向多家机构申请贷款 | 人工复核 |
| 团伙欺诈 | 短时间内相同设备/地址集中申请 | 拒绝 |
| 恶意申贷 | 有被执行信息、行政处罚 | 人工复核 |

#### 3.1.3 额度测算规则

**个人消费贷款**:
```
额度 = min(收入×倍数, 产品上限, 负债能力上限)
收入倍数：通常为月收入的12-36倍
```

**企业经营贷款**:
```
额度 = min(年营收×比例, 抵押物评估值×抵押率, 现金流覆盖)
年营收比例：通常为20%-30%
抵押率：住宅70%、商铺50%、工业厂房60%
```

### 3.2 贷中风控规则

#### 3.2.1 监控规则

| 监控类型 | 监控内容 | 触发阈值 | 处置措施 |
|----------|----------|----------|----------|
| 交易异常 | 大额资金进出、频繁转账 | 单笔>50%额度 | 预警 |
| 行为异常 | 设备更换、地址变更 | 短时间内多次 | 预警 |
| 还款异常 | 提前还款、规律变更 | 偏离历史规律 | 关注 |
| 外部风险 | 诉讼信息、行政处罚 | 发现新增 | 复核 |

#### 3.2.2 额度调整规则

| 调整类型 | 触发条件 | 调整幅度 | 上限 |
|----------|----------|----------|------|
| 提额 | 良好还款记录6个月以上 | +10%~30% | 产品上限 |
| 降额 | 连续逾期或负债上升 | -20%~50% | 剩余本金 |
| 冻结 | 严重逾期或欺诈嫌疑 | 100% | - |

#### 3.2.3 风险预警规则

| 预警级别 | 触发条件 | 响应时间 | 处置措施 |
|----------|----------|----------|----------|
| 红色预警 | 逾期≥30天或诉讼 | 24小时内 | 冻结额度、启动催收 |
| 橙色预警 | 逾期15-29天 | 3天内 | 加大监控、联系客户 |
| 黄色预警 | 逾期7-14天 | 7天内 | 提醒还款、关注 |

### 3.3 贷后风控规则

#### 3.3.1 催收管理规则

| 逾期阶段 | 催收方式 | 催收频率 | 处置策略 |
|----------|----------|----------|----------|
| M1（1-30天） | 短信、电话提醒 | 每日 | 提醒还款、核对信息 |
| M2（31-60天） | 电话、上门 | 每周2-3次 | 了解原因、协商还款 |
| M3（61-90天） | 电话、外访、函件 | 每日 | 施加压力、告知后果 |
| M4+（90天以上） | 诉讼、处置抵押物 | 持续 | 法律诉讼、资产处置 |

#### 3.3.2 诉讼时效规则

| 诉讼类型 | 时效期限 | 注意事项 |
|----------|----------|----------|
| 民事诉讼 | 3年（自还款到期日起算） | 中断时效需保留证据 |
| 申请执行 | 2年（判决生效后） | 及时申请避免失效 |
| 担保责任 | 主债务履行期届满6个月 | 担保人需及时主张 |

#### 3.3.3 不良处置规则

| 处置方式 | 适用情况 | 处置周期 |
|----------|----------|----------|
| 债务重组 | 有还款意愿但暂时困难 | 30-60天 |
| 核销 | 无法回收的损失类贷款 | 按监管要求 |
| 打包转让 | 大批量不良资产 | 90-180天 |
| 诉讼清收 | 有资产可执行 | 6-24个月 |
| 以物抵债 | 无法回收现金 | 评估后处置 |

### 3.4 组合管理规则

#### 3.4.1 集中度管理

| 集中度类型 | 监管要求 | 行内限制 |
|------------|----------|----------|
| 单户集中度 | ≤10%（单一客户贷款余额/资本净额） | ≤8% |
| 集团集中度 | ≤15% | ≤12% |
| 行业集中度 | 适度分散 | 单一行业≤30% |
| 地区集中度 | 适度分散 | 单一地区≤40% |

#### 3.4.2 资产组合监控

| 监控指标 | 目标值 | 预警阈值 |
|----------|--------|----------|
| 不良率 | ≤1.5% | >2% |
| 逾期率 | ≤3% | >5% |
| 损失率 | ≤0.5% | >1% |
| 拨备覆盖率 | ≥150% | <120% |

## 四、风险评估模型

### 4.1 评分卡模型

#### 4.1.1 评分卡构成

| 评分维度 | 权重 | 核心变量 |
|----------|------|----------|
| 基础属性 | 20% | 年龄、学历、婚姻状态、地区 |
| 信用历史 | 30% | 征信查询次数、历史逾期次数、贷款余额 |
| 收入稳定性 | 25% | 工作年限、单位性质、收入水平、社保缴纳 |
| 负债情况 | 15% | 负债率、负债类型、在贷机构数 |
| 行为数据 | 10% | APP活跃度、登录频次、填写行为轨迹 |

#### 4.1.2 评分卡示例（简化版）

| 变量 | 分箱 | 评分 |
|------|------|------|
| 年龄 | 25-35岁 | +20 |
| | 36-45岁 | +15 |
| | 46-55岁 | +10 |
| 征信逾期 | 无逾期 | +25 |
| | 1-2次逾期 | +10 |
| | 3次以上 | -15 |
| 负债率 | ≤30% | +20 |
| | 31-50% | +10 |
| | >50% | -10 |
| 工作年限 | ≥3年 | +15 |
| | 1-3年 | +10 |
| | <1年 | +5 |

**总分 = 各变量评分之和 + 基准分（400）**

### 4.2 风险定价模型

#### 4.2.1 PD-LGD-EAD 模型

| 参数 | 定义 | 估计方法 |
|------|------|----------|
| PD（违约概率） | 借款人未来一年违约的概率 | 历史违约率统计、迁移矩阵 |
| LGD（违约损失率） | 违约后的损失比例 | 清收数据统计、回收率模型 |
| EAD（违约敞口） | 违约时的暴露金额 | 余额预测、提取规则 |

**预期损失 (EL) = PD × LGD × EAD**

#### 4.2.2 定价公式

```
贷款利率 = 资金成本 + 运营成本 + 预期损失 + 目标利润率
        = FTP + OC + (PD × LGD × EAD) / EAD + Margin
```

### 4.3 机器学习模型

#### 4.3.1 常用算法

| 算法 | 适用场景 | 优缺点 |
|------|----------|--------|
| 逻辑回归 | 基础评分卡、解释性强 | 精度有限、难以捕捉非线性 |
| 决策树 | 规则提取、可解释性好 | 容易过拟合、稳定性差 |
| 随机森林 | 精度要求高 | 解释性较弱 |
| XGBoost/LightGBM | 大数据量、高精度 | 调参复杂、需大量样本 |
| 神经网络 | 复杂模式识别 | 黑箱、需专业团队 |

#### 4.3.2 模型评估指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| AUC | 模型区分能力 | ≥0.75 |
| KS | 最大区分度 | ≥0.4 |
| PSI | 稳定性 | ≤0.1 |
| 准确率 | 预测正确比例 | 因场景而异 |
| 召回率 | 违约客户被识别比例 | ≥0.6 |
| F1 Score | 精确率与召回率调和均值 | ≥0.6 |

### 4.4 欺诈检测模型

#### 4.4.1 欺诈类型

| 类型 | 特征 | 检测方法 |
|------|------|----------|
| 身份欺诈 | 冒用他人身份、信息造假 | 人脸识别、活体检测、证件比对 |
| 账户欺诈 | 盗用账户、账户共享 | 设备指纹、行为生物特征 |
| 交易欺诈 | 虚假交易、套现 | 交易模式分析、关联网络 |
| 团伙欺诈 | 有组织欺诈行为 | 知识图谱、社交网络分析 |

#### 4.4.2 欺诈评分

```
欺诈风险分 = 设备风险分 × 30% + 行为风险分 × 30% + 
             关联风险分 × 25% + 外部风险分 × 15%

阈值规则：
- ≥80分：直接拒绝
- 60-79分：人工复核
- <60分：通过
```

## 六、法规体系

### 6.1 核心法律法规

| 法规名称 | 效力级别 | 施行日期 | 核心内容 |

| 《中华人民共和国银行业监督管理法》 | 法律 | 2007.1.1 | 银行业监管职责、行政处罚 |

| 《商业银行资本管理办法（试行）》 | 银保监会规章 | 2013.1.1 | 资本充足率、风险加权资产 |
| 《商业银行信息披露办法》 | 银保监会规章 | 2007.7.3 | 信息披露要求、市场约束 |
| 《网络借贷信息中介机构业务活动管理暂行办法》 | 银保监会规章 | 2016.8.17 | P2P监管、业务规范 |

### 6.2 监管指标

| 指标名称 | 监管要求 | 计算公式 |
|----------|----------|----------|
| 资本充足率 | ≥8% | 资本净额 / 风险加权资产 × 100% |
| 核心一级资本充足率 | ≥4.5% | 核心一级资本 / 风险加权资产 × 100% |
| 不良贷款率 | ≤5% | 不良贷款余额 / 贷款总余额 × 100% |
| 拨备覆盖率 | ≥150% | 贷款损失准备 / 不良贷款余额 × 100% |
| 贷款集中度 | ≤10% | 单一客户贷款余额 / 资本净额 × 100% |
| 流动性比例 | ≥25% | 流动性资产 / 流动性负债 × 100% |

## 七、应用场景

1. **智能信贷审批**: 基于本体规则自动判断准入、额度、定价
2. **欺诈检测**: 利用规则引擎和机器学习模型识别欺诈行为
3. **风险预警**: 实时监控客户行为，触发预警并自动处置
4. **催收管理**: 根据逾期阶段匹配催收策略，优化资源配置
5. **组合监控**: 监控资产组合风险指标，辅助管理决策
6. **合规检查**: 自动检查业务是否符合监管要求
