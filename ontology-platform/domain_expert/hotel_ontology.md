# 酒店领域本体设计

## 概述

本文档定义酒店领域的核心本体模型，涵盖酒店运营中的核心实体、关系、业务规律和规则。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 酒店 | Hotel | 提供住宿服务的设施 | 酒店ID、名称、星级、位置、房量 |
| 客房 | Room | 酒店的基本住宿单元 | 房号、房型、面积、床型、价格 |
| 客人 | Guest | 酒店的客户 | 客人ID、姓名、联系方式、入住历史 |
| 预订 | Reservation | 客人的住宿预订 | 预订号、入住日期、退房日期、房型 |
| 入住 | Check-in | 客人办理入住 | 入住号、客人ID、房号、入住时间 |
| 退房 | Check-out | 客人办理退房 | 退房号、入住ID、退房时间、费用 |
| 房价 | Room Rate | 客房价格 | 房型、价格、日期范围、适用渠道 |
| 房型 | Room Type | 客房类型 | 类型名称、床型、面积、可住人数 |
| 会员 | Member | 酒店会员 | 会员ID、等级、积分、权益 |
| 员工 | Staff | 酒店员工 | 员工ID、姓名、岗位、部门 |

### 1.2 关系定义

```
酒店 ⊚ 拥有 [客房] (一对多)
客房 ⊚ 属于 [房型] (多对一)
客人 ⊚ 进行 [预订] (一对多)
预订 ⊚ 关联 [客房] (多对一)
客人 ⊚ 办理 [入住] (一对多)
入住 ⊚ 触发 [退房] (一对一)
客人 ⊚ 注册 [会员] (一对一)
酒店 ⊚ 制定 [房价] (一对多)
```

## 二、规律 (Laws)

### 2.1 客房收益管理规律

**公式**:
```
RevPAR = 实际总收入 / 可售房晚数
OCC = 实际入住房晚数 / 可售房晚数 × 100%
ADR = 实际总收入 / 已售房晚数
```

**行业基准**:
- 经济型: OCC 60-70%, ADR 150-250元
- 中端: OCC 65-75%, ADR 300-500元
- 高端: OCC 60-80%, ADR 600-1500元
- 奢华: OCC 55-75%, ADR 1500+

### 2.2 入住率波动规律

- **周一至周四**: 商旅客为主，入住率较低
- **周五至周日**: 休闲客为主，入住率较高
- **节假日**: 入住率可达100%，价格上浮50-200%

## 三、业务规则 (Rules)

### 3.1 预订规则

| 规则 | 说明 |
|------|------|
| 预订保留 | 18:00前到店，超时自动取消 |
| 预订取消 | 提前24h免费取消 |
| 预付款 | 旺季或长住需预付定金 |
| 会员优先 | 会员预订优先确认 |

### 3.2 入住规则

| 规则 | 说明 |
|------|------|
| 入住时间 | 14:00后入住，12:00前退房 |
| 证件登记 | 必须出示身份证件登记 |
| 押金 | 需缴纳住房押金 |
| 加床 | 加床费用100-200元/晚 |

### 3.3 房价规则

```
基础价 → 渠道折扣 → 会员折扣 → 活动优惠 → 最终价格
```

| 渠道 | 折扣比例 |
|------|----------|
| 官网 | 95折 |
| OTA | 无折扣 |
| 协议价 | 8折 |
| 会员价 | 等级决定 |

### 3.4 会员等级规则

| 等级 | 年消费额 | 积分倍率 | 权益 |
|------|----------|----------|------|
| 普通 | <5000 | 1x | 基本折扣 |
| 银卡 | 5000-15000 | 1.5x | 延迟退房 |
| 金卡 | 15000-50000 | 2x | 免费升房 |
| 钻石 | >50000 | 3x | 专属服务 |

## 四、应用场景

1. **智能定价**: 基于入住率、日期、渠道动态调整房价
2. **会员运营**: 积分兑换、等级维护、流失预警
3. **收益管理**: RevPAR优化、房型组合策略
4. **服务自动化**: 入住提醒、续住推荐、投诉处理
