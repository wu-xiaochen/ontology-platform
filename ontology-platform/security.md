# 安全配置

本文档定义了本体论平台的安全策略、审计日志规范和数据加密标准。

---

## 1. 安全策略

### 1.1 访问控制

- **最小权限原则**：所有用户和系统组件仅授予完成任务所需的最小权限
- **角色基于访问控制 (RBAC)**：根据用户角色分配权限，定期审查角色权限
- **多因素认证 (MFA)**：敏感操作必须启用多因素认证
- **会话管理**：
  - 会话超时：空闲 30 分钟后自动登出
  - 并发会话限制：同一账户最多 3 个并发会话

### 1.2 网络安全

- **传输层安全**：所有 HTTP 流量必须使用 TLS 1.2+ 加密
- **API 安全**：
  - API 密钥轮换：每 90 天轮换一次
  - 请求速率限制：每个 IP 每分钟最多 100 次请求
  - 禁止敏感数据通过 URL 参数传递
- **防火墙**：仅开放必要端口，禁用不必要的服务

### 1.3 数据保护

- **敏感数据分类**：
  - 高度敏感：用户密码、金融数据、生物识别信息
  - 中度敏感：个人身份信息、联系方式
  - 低度敏感：公开信息、日志数据
- **数据脱敏**：日志和调试输出中必须脱敏处理敏感字段
- **数据保留**：根据合规要求设置数据保留期限，自动清理过期数据

### 1.4 漏洞管理

- **定期扫描**：每月执行一次漏洞扫描
- **补丁管理**：严重漏洞 72 小时内修复，高危漏洞 7 天内修复
- **依赖安全**：定期更新第三方依赖，检查已知漏洞

## 2. 审计日志

### 2.1 日志记录要求

必须记录以下事件：

| 事件类型 | 记录内容 | 保留期限 |
|---------|---------|---------|
| 认证事件 | 登录、登出、认证失败、MFA 验证 | 2 年 |
| 授权事件 | 权限变更、角色分配、资源访问拒绝 | 2 年 |
| 数据操作 | 创建、读取、更新、删除敏感数据 | 2 年 |
| 管理操作 | 系统配置变更、用户管理、API 密钥操作 | 5 年 |
| 安全事件 | 异常登录、攻击检测、漏洞发现 | 5 年 |

### 2.2 日志格式

```json
{
  "timestamp": "2024-01-15T10:30:00.000Z",
  "level": "INFO",
  "event_type": "AUTH_LOGIN",
  "user_id": "user_12345",
  "ip_address": "192.168.1.100",
  "user_agent": "Mozilla/5.0...",
  "result": "SUCCESS",
  "mfa_used": true,
  "details": {
    "login_method": "password",
    "session_id": "sess_abc123"
  }
}
```

### 2.3 日志保护

- **完整性**：使用哈希链确保日志不可篡改
- **加密存储**：敏感日志字段加密存储
- **访问控制**：仅安全管理员可访问原始日志
- **备份**：日志每日备份，保留 90 天

### 2.4 日志监控

- **实时告警**：失败登录超过 5 次 / 10 分钟、异常访问模式
- **定期审计**：每月生成安全审计报告
- **合规检查**：确保满足 GDPR、ISO 27001 等合规要求

## 3. 数据加密

### 3.1 加密标准

- **算法**：AES-256-GCM（对称加密）、RSA-4096 或 ECDSA P-384（非对称加密）
- **密钥管理**：
  - 使用硬件安全模块 (HSM) 或云 KMS 存储密钥
  - 密钥分离：数据加密密钥 (DEK) 与密钥加密密钥 (KEK) 分离
- **密钥轮换**：加密密钥每年轮换一次，敏感密钥每 90 天轮换

### 3.2 数据加密场景

| 场景 | 加密方式 | 密钥类型 |
|-----|---------|---------|
| 存储数据 (at rest) | AES-256-GCM | DEK |
| 网络传输 (in transit) | TLS 1.2+ | 证书密钥 |
| 敏感字段 | AES-256-GCM | 字段级 KEK |
| 备份数据 | AES-256-GCM | 备份专用 KEK |
| API 密钥/令牌 | AES-256-GCM | 保险库密钥 |

### 3.3 密钥管理流程

1. **密钥生成**：使用安全随机数生成器
2. **密钥存储**：加密后存储在 HSM/KMS，原始密钥不持久化
3. **密钥使用**：密钥在内存中解密，使用后立即清除
4. **密钥销毁**：密钥撤销后安全擦除，无法恢复

### 3.4 敏感数据加密示例

```python
# 字段级加密示例
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import base64

class FieldEncryptor:
    def __init__(self, key: bytes):
        self.aesgcm = AESGCM(key)
    
    def encrypt(self, plaintext: str, nonce: bytes) -> str:
        ciphertext = self.aesgcm.encrypt(nonce, plaintext.encode(), None)
        return base64.b64encode(nonce + ciphertext).decode()
    
    def decrypt(self, encrypted: str) -> str:
        data = base64.b64decode(encrypted)
        nonce, ciphertext = data[:12], data[12:]
        return self.aesgcm.decrypt(nonce, ciphertext, None).decode()
```

## 4. 应急响应

### 4.1 安全事件分级

- **P0 (紧急)**：数据泄露、系统瘫痪、勒索攻击 → 1 小时内响应
- **P1 (高)**：账户被盗、未授权访问 → 4 小时内响应
- **P2 (中)**：异常行为、可疑活动 → 24 小时内响应
- **P3 (低)**：轻微违规、配置问题 → 7 天内响应

### 4.2 响应流程

1. **检测**：安全监控系统告警
2. **评估**：确定事件范围和影响
3. **遏制**：隔离受影响系统，阻断攻击
4. **根除**：清除恶意软件/后门，修复漏洞
5. **恢复**：恢复系统至正常状态
6. **复盘**：编写事件报告，优化防护措施

## 5. 合规要求

- **GDPR**：个人数据处理遵循最小化原则，保障数据主体权利
- **ISO 27001**：信息安全管理体系符合国际标准
- **SOC 2**：安全性、可用性、保密性、隐私性控制
