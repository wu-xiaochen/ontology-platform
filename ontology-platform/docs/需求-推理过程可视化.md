## 1. 需求背景

在ontology-clawra skill的使用过程中，用户确认了一种清晰直观的推理展示方式，并将此作为ontology-platform的核心功能需求。

---

## 2. 功能需求

### 2.1 推理过程全链路展示

| 模块 | 功能描述 |
|------|----------|
| **输入条件** | 展示用户的原始输入条件 |
| **规则匹配** | 展示匹配到的Law（法规/场景）和Rule（价格/配置规则） |
| **推理链路图** | 可视化展示推理过程 |
| **置信度标注** | 每一步结论标注CONFIRMED/ASSUMED/SPECULATIVE/UNKNOWN |

### 2.2 本体库分层管理

| 本体库 | 存储位置 | 同步策略 |
|--------|----------|----------|
| 专业领域库 | skills/ontology-clawra/memory/ | Git管理，但.gitignore保护隐私数据 |
| 工作学习库 | workspace/memory/ontology/ | 本地私有，不上传 |
| 项目需求库 | workspace/docs/ | Git管理 |

### 2.3 推理引擎核心能力

```python
# 伪代码：推理流程
def reason(input_conditions):
    # 1. 条件解析
    conditions = parse(input_conditions)
    
    # 2. 规则匹配
    matched_laws = search_laws(conditions)  # 法规/场景
    matched_rules = search_rules(conditions)  # 配置/价格
    
    # 3. 置信度评估
    confidence = evaluate_confidence(matched_laws, matched_rules)
    
    # 4. 推理链路生成
    reasoning_chain = build_chain(conditions, matched_laws, matched_rules)
    
    # 5. 结论输出
    return {
        "matched_laws": matched_laws,
        "matched_rules": matched_rules,
        "reasoning_chain": reasoning_chain,
        "confidence": confidence,
        "recommendations": generate_recommendations(reasoning_chain)
    }
```

## 3. 非功能需求

### 3.1 性能
- 单次推理响应时间 < 2秒
- 支持1000+规则的快速匹配

### 3.2 可扩展性
- 支持插件式规则引擎
- 支持多领域本体库切换

### 3.3 可解释性
- 每一步推理必须有明确依据
- 支持追溯到原始Law/Rule
