## 📌 愿景

构建企业级**垂直领域可信AI推理引擎平台**，解决AI幻觉问题，通过结构化本体推理 + 置信度追踪 + 自动学习机制，为供应链、医疗、金融、制造等垂直领域提供可解释、可追溯的智能决策支持。

## 🚀 M1: 核心推理引擎 (Q2 2026)

### 目标
完成推理引擎核心功能，支持前向/后向链推理，提供RESTful API和GraphQL接口。

### 交付物

- [ ] **推理引擎 v1.0**
  - [x] 前向链推理 (forward chaining)
  - [x] 后向链推理 (backward chaining)
  - [x] 混合推理策略
  - [x] 置信度传播算法

- [ ] **RESTful API**
  - [ ] `/api/v1/query` - 本体查询接口
  - [ ] `/api/v1/reason` - 推理执行接口
  - [ ] `/api/v1/validate` - 断言验证接口
  - [ ] `/api/v1/explain` - 推理解释接口

- [ ] **GraphQL API**
  - [ ] Query: `queryOntology`, `reason`, `explain`
  - [ ] Mutation: `addFact`, `addRule`
  - [ ] Subscription: `reasoningProgress`

- [ ] **推理可解释性**
  - [ ] 推理路径可视化
  - [ ] 证据链展示
  - [ ] 置信度分级标注 (CONFIRMED/ASSUMED/SPECULATIVE/UNKNOWN)

### 依赖
- Python 3.9+
- FastAPI
- GraphQL ( Strawberry / Graphene )

## 🚀 M1.5: 自动学习机制 (Q2 2026)

### 目标
实现基于用户反馈的自动本体学习功能，完成从被动到主动的范式转换。

### 核心升级

| v3.2 (旧版) | v3.5 (新版) |
|------------|-------------|
| 用户不说"抽取" → 不提取 | 用户确认推理 → **自动提取** |
| 重复概念 → 忽略 | 重复出现3次 → **自动识别** |
| 推理失败 → 等待用户问 | 推理失败 → **主动建议补充** |

### 交付物

- [ ] **自动学习引擎**
  - [ ] 用户确认推理结果后自动抽取
  - [ ] 高频实体自动识别（阈值3次）
  - [ ] 推理失败时主动建议补充本体
  - [ ] 用户纠正错误时自动更新

- [ ] **置信度自动升级**
  - [ ] ASSUMED → CONFIRMED (用户确认后)
  - [ ] 置信度追踪与历史记录
  - [ ] 置信度趋势可视化

- [ ] **智能触发器**
  - [ ] `user_confirms_reasoning` 事件处理
  - [ ] `entity_mentioned_3_times` 事件处理
  - [ ] `ontology_lookup_failed` 事件处理
  - [ ] `user_correction` 事件处理

- [ ] **增量更新机制**
  - [ ] 避免覆盖式写入
  - [ ] 去重检查（名称匹配 + 相似度）
  - [ ] 变更来源记录

### 技术实现
- 事件驱动架构
- 增量更新算法
- 相似度匹配 (threshold: 0.8)
