# 核心概念澄清：本体 (Ontology) vs 知识图谱 (Knowledge Graph)

## 1. 辩证关系

在 Clawra 认知引擎中，这两者并非平级替代关系，而是**模式**与**实例**的关系：

| 维度 | 本体 (Ontology) | 知识图谱 (Knowledge Graph) |
| :--- | :--- | :--- |
| **层级** | **模式层 (T-Box)** | **数据层 (A-Box)** |
| **类比** | 法律 / 交通规则 | 具体的行为 / 车流 |
| **核心定义** | 类(Class)、属性(Property)、公理(Axiom) | 实例(Individual)、事实(Fact) |
| **例子** | "所有'调压箱'都必须具备'安全等级'" | "型号 RTX-100 的调压箱其安全等级是甲级" |
| **逻辑作用** | 提供推理的公理依据（如果你是 A，那么你必然是 B） | 提供推理的原材料 |

---

## 2. 为什么现在的实现是"本体知识图谱"？

传统的知识图谱往往只是数据的堆砌（点和边）。而我们的系统引入了 **OWL 语义约束**：
- **约束驱动**：如果你试图存入一个违反本体定义的事实（例如给一个'非调压箱'实体添加'进口压力'属性），**Sentinel（哨兵）** 会根据本体规则拦截它。
- **推理驱动**：**Reasoner（推理引擎）** 并不是在简单查表，而是在利用本体定义的**传递性、对称性、等价性**来发现隐含事实。

---

## 3. Palantir 带来的启示：从静态到动力学 (Kinetic)

您提到的 Palantir 本体论核心在于它不满足于静态的"描述"，而是引入了 **Action Types**：
- **静态层**：Object + Link (即我们现在的图谱)。
- **动态层 (Kinetic)**：**Action**。它定义了如何改变这个图，以及改变时需要满足的复杂逻辑校验。

**Clawra 演进方向**：我们正在将架构从纯粹的描述性 OWL 转向支持 `execute_action` 的动力学模型，使 Agent 能够执行任务（如：自动闭环合规核对），而不仅仅是回答问题。
